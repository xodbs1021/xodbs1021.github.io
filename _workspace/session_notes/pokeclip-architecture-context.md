# PokeClip 아키텍처 설계 경위

## 문제
치지직 방송하면서 게임/마이크/디스코드/후원 오디오를 트랙별로 분리 분석 필요.
RTMP는 단일 오디오 트랙만 지원 → 분석용 별도 스트림 필요.

## 설계 과정

### 1차 접근
- `obs-multi-rtmp` 오픈소스 플러그인 발견 (https://github.com/sorayuki/obs-multi-rtmp)
- 다중 RTMP 동시 송출 플러그인을 포크해서 한쪽은 RTMP, 한쪽은 SRT+MPEG-TS로 수정하려 했음

### 방향 전환
- OBS가 네이티브로 멀티 출력 지원한다는 걸 발견
- 플러그인 수정 없이 OBS 설정에서 SRT 출력 바로 추가 가능
- 외부 의존 없이 더 안정적 → 네이티브 방향으로 전환

## 최종 구조
OBS 네이티브 멀티 출력:
- RTMP → 치지직 (방송용, 단일 믹스 오디오)
- SRT+MPEG-TS → PokeClip EC2 (분석용, 멀티 오디오 트랙)
