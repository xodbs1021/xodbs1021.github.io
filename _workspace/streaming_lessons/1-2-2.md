## 레벨 1-2 / 2강: 비트레이트 — 화질의 실제 결정 요인

---

### 비트레이트가 뭔가

1초에 전송되는 데이터 양. 단위는 **bps (bits per second)**.

```
1 Kbps = 1,000 bps
1 Mbps = 1,000,000 bps
```

치지직 1080p60 권장이 6,000 Kbps = 6 Mbps. 1초에 6백만 비트, 약 750KB의 영상 데이터가 흐른다.

---

### 해상도보다 비트레이트가 더 중요한 경우

같은 1080p인데 화질이 다르게 보이는 영상을 본 적 있을 거다. 비트레이트 차이다.

```
1080p @ 6 Mbps: 깨끗함
1080p @ 2 Mbps: 빠른 움직임에서 깍두기(블록 노이즈) 보임
1080p @ 0.5 Mbps: 거의 안 봄. 뭉개진 1080p
```

해상도는 픽셀의 "수"고, 비트레이트는 각 픽셀에 쓸 수 있는 "정보량"이다. 비트레이트가 낮으면 픽셀 수가 많아도 각 픽셀에 줄 정보가 부족해서 압축 손실이 커진다.

**같은 비트레이트라면 낮은 해상도가 더 깨끗할 수 있다.**

```
2 Mbps:
  - 1080p로 인코딩 → 픽셀당 정보 부족 → 깍두기
  - 720p로 인코딩 → 픽셀 수 적어 정보 충분 → 깨끗

→ 1080p보다 720p가 더 좋아 보임
```

치지직이 모바일 시청자에게 720p나 480p를 주는 게 단순히 데이터 절약 때문만이 아니다. 비트레이트가 낮은 상황에서는 낮은 해상도가 실제로 더 좋아 보인다.

---

### CBR vs VBR — 비트레이트를 어떻게 분배하나

영상은 장면마다 압축 난이도가 다르다.

```
고정된 토크쇼 화면: 변화 적음 → 압축 잘 됨 → 비트 적게 써도 됨
폭발 장면 / 빠른 카메라 워크: 변화 많음 → 비트 많이 필요
```

**CBR (Constant Bit Rate)**: 비트레이트 고정. 1초마다 정확히 6 Mbps.
- 장점: 대역폭 예측 가능. 라이브에 유리
- 단점: 쉬운 장면엔 낭비, 어려운 장면엔 부족

**VBR (Variable Bit Rate)**: 비트레이트 가변. 평균 6 Mbps 목표, 어려운 장면엔 8 Mbps, 쉬운 장면엔 3 Mbps.
- 장점: 화질이 일정
- 단점: 순간 대역폭이 튀어서 라이브 송출 시 불안정

라이브 스트리밍은 보통 **CBR**을 쓴다. 네트워크 대역폭을 예측 가능하게 유지하기 위해서. VOD는 VBR이 일반적이다.

OBS 설정 화면에서 본 적 있는 그 옵션이 이거다.

---

### 비트레이트 인코딩 모드 — CRF, CQ

VBR의 변형 중 화질 우선 모드들도 있다.

**CRF (Constant Rate Factor)**: x264/x265에서 쓰는 모드. 비트레이트 대신 "품질 점수"를 고정한다.

```
ffmpeg -i input.mp4 -c:v libx264 -crf 23 output.mp4
```

CRF 23이 기본값. 18은 거의 무손실, 28은 낮은 화질. 비트레이트는 영상에 따라 자동으로 결정된다.

라이브에선 못 쓴다. 출력 비트레이트를 알 수 없으니 네트워크 대역폭 계획이 안 선다. VOD 인코딩에 쓰는 모드다.

---

### 비트레이트와 인코딩 시간의 관계

같은 비트레이트라도 인코더가 얼마나 열심히 압축하느냐에 따라 화질이 달라진다.

FFmpeg `-preset` 옵션:

```
ultrafast: 빠름, 같은 비트레이트에서 화질 낮음
veryfast:  라이브 기본값
medium:    VOD 기본값
slow:      느림, 같은 비트레이트에서 화질 높음
veryslow:  최고 화질
```

라이브는 실시간 처리가 필요해서 `veryfast` 정도가 한계다. medium 이상을 쓰면 인코딩이 재생을 따라가지 못한다.

VOD는 한 번 인코딩하고 끝이니 `slow`를 써서 같은 파일 크기에 더 좋은 화질을 뽑는다.

---

### NVENC — GPU 인코딩이 다른 점

x264는 CPU 인코딩이다. CPU가 비트 하나하나 계산한다. 정밀하지만 느리다.

NVENC는 NVIDIA GPU의 전용 인코딩 칩을 쓴다. 압도적으로 빠르지만 같은 비트레이트에서 x264보다 화질이 약간 낮다.

```
같은 비트레이트 6 Mbps에서:
- x264 medium: 화질 최상, CPU 50% 사용
- NVENC:       화질 약간 낮음, CPU 1% 사용
```

라이브 스트리머가 NVENC를 쓰는 이유는 게임이 CPU를 써야 하기 때문이다. 인코딩에 CPU를 뺏기면 게임 프레임이 떨어진다.

치지직 같은 플랫폼 서버 측에서 NVENC를 쓰는 이유는 비용이다. 1차 멘토 피드백에 "NVENC GPU 트랜스코딩으로 비용을 60% 절감"이 있던 게 이거다.

---

### 2강 핵심 3줄

1. 비트레이트는 1초당 데이터량 — 같은 해상도라도 비트레이트가 낮으면 깍두기가 보이고, 낮은 비트레이트에선 720p가 1080p보다 깨끗할 수 있다
2. CBR은 대역폭 예측 가능해서 라이브에 적합, VBR/CRF는 화질 일정해서 VOD에 적합
3. NVENC는 GPU 인코딩 — 같은 비트레이트에서 x264보다 화질 약간 낮지만 CPU 안 써서 라이브에 필수
