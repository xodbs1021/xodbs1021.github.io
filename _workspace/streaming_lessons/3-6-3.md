### 흔히 만나는 코덱 문제

라이브 플랫폼 운영 시 자주 발생.

```
[자주 만나는 문제]
1. 시청자가 영상 못 봄
2. 음성만 들리고 영상 안 보임 (또는 반대)
3. 화질이 갑자기 떨어짐
4. 지연이 크게 늘어남
5. 일부 디바이스만 안 됨
6. 색상이 이상함 (HDR 관련)
7. 입 모양과 소리 안 맞음 (AV Sync)
8. 화면 깨짐 (artifact)
```

각 문제의 원인과 진단 방법.

---

### 진단 도구 1 - FFprobe

영상 파일 분석의 기본.

```bash
# 기본 정보
ffprobe input.mp4

# 상세 JSON
ffprobe -v error -show_streams -show_format -of json input.mp4

# 비디오 정보만
ffprobe -v error -select_streams v:0 \
  -show_entries stream=codec_name,profile,level,width,height,r_frame_rate,pix_fmt,bit_rate \
  -of csv=p=0 input.mp4

# 오디오 정보만
ffprobe -v error -select_streams a:0 \
  -show_entries stream=codec_name,profile,sample_rate,channels,bit_rate \
  -of csv=p=0 input.mp4
```

출력 예시:
```
h264,High,40,1920,1080,30/1,yuv420p,6000000
aac,LC,48000,2,128000
```

이걸로 받은 영상의 코덱 사양 확인.

### 진단 도구 2 - 브라우저 개발자 도구

웹 시청자 측 디버깅.

```javascript
// 비디오 요소 확인
const video = document.querySelector('video');
console.log('Current src:', video.src);
console.log('Ready state:', video.readyState);
console.log('Network state:', video.networkState);

// 에러 확인
video.addEventListener('error', (e) => {
  console.error('Video error:', video.error);
});

// 화질/코덱 정보 (MediaCapabilities API)
const capabilities = await navigator.mediaCapabilities.decodingInfo({
  type: 'media-source',
  video: {
    contentType: 'video/mp4; codecs="avc1.640028"',
    width: 1920,
    height: 1080,
    bitrate: 6000000,
    framerate: 60,
  },
});
console.log('H.264 1080p60 supported:', capabilities.supported);
console.log('Smooth:', capabilities.smooth);
console.log('Power efficient:', capabilities.powerEfficient);
```

이걸로 시청자 디바이스가 코덱 지원하는지 확인.

### 진단 도구 3 - hls.js 디버깅

HLS 시청자 측.

```javascript
const hls = new Hls({
  debug: true,  // 모든 이벤트 로그
});

// 코덱 관련 에러
hls.on(Hls.Events.ERROR, (event, data) => {
  if (data.fatal) {
    switch(data.type) {
      case Hls.ErrorTypes.MEDIA_ERROR:
        console.error('Media error - 코덱 문제 가능');
        hls.recoverMediaError();
        break;
      case Hls.ErrorTypes.NETWORK_ERROR:
        console.error('Network error');
        break;
    }
  }
});

// 코덱 정보
hls.on(Hls.Events.MANIFEST_PARSED, (event, data) => {
  console.log('Levels:', data.levels);
  data.levels.forEach((level, idx) => {
    console.log(`Level ${idx}: ${level.codecs} ${level.width}x${level.height} ${level.bitrate}`);
  });
});
```

### 문제 1: 영상 안 보임

```
[증상]
페이지 로드되지만 video 태그가 검은 화면

[원인 후보]
A. 코덱 미지원
B. 코덱 문자열 잘못
C. 파일 손상
D. CORS 문제
E. DRM 인증 실패

[진단 순서]
1. 브라우저 개발자 도구 콘솔 확인
2. video.error 객체 확인
3. ffprobe로 파일 분석
4. MediaCapabilities로 디바이스 지원 확인
5. CORS 헤더 확인

[해결]
H.264로 폴백 트랙 제공
또는 코덱 문자열 정확히 (avc1.640028 등)
```

```

### 문제 3: 화질 갑자기 떨어짐

```
[증상]
1080p로 시작했다가 갑자기 360p

[원인]
ABR 알고리즘이 대역폭 부족 감지
네트워크 일시적 끊김
서버 측 트랜스코딩 문제

[진단]
hls.js의 LEVEL_SWITCHED 이벤트 추적

시청자 네트워크 측정 (navigator.connection)

[해결]
ABR 알고리즘 튜닝 (abrBandWidthUpFactor 등)
화질 floor 설정 (최소 480p 보장)
CDN 응답 시간 점검
```

### 문제 5: 일부 디바이스만 안 됨

```
[증상]
PC는 정상, iPhone에서 안 됨

[원인]

H.265 hvc1 vs hev1 태그 (iOS는 hvc1만)

[진단]

디바이스별 실패 비율 통계

[해결]
iOS는 HLS 강제
H.265는 -tag:v hvc1
오디오는 AAC 폴백 트랙
```

### 문제 6: 색상 이상 (HDR)

```
[증상]
HDR 영상이 SDR 디스플레이에서 색 이상
또는 매우 어둡거나 채도 이상

[원인]

[진단]
ffprobe -show_frames | grep color_

[해결]
HDR → SDR 변환 시 적절한 tone mapping

서버 측 SDR 트랙 따로 제공
```

### 문제 7: AV Sync 어긋남

```
[증상]
입 모양과 음성이 안 맞음

### 문제 8: 화면 깨짐 (Artifact)

```
[증상]
모자이크, 색번짐, 블록 현상

[원인]
비트레이트 부족
인코더 preset 너무 빠름
키프레임 누락
패킷 손실

[진단]
ffmpeg -i input.mp4 -vf "select=eq(pict_type\,I)" -vsync vfr keyframes/%04d.png
키프레임 위치 확인

[해결]
비트레이트 올림
인코더 preset 한 단계 느리게 (veryfast → faster)
GOP 짧게 (60 → 30)
SRT/QUIC 같은 손실 복구 프로토콜
```

### 로그 분석 - FFmpeg

인코딩 로그에서 문제 찾기.

```bash
# 상세 로그
ffmpeg -loglevel debug -i input.mp4 output.mp4 2> debug.log

# 통계만
ffmpeg -loglevel info -stats -i input.mp4 output.mp4

# 에러만
ffmpeg -loglevel error -i input.mp4 output.mp4
```

로그에서 찾을 키워드:
```
Bitstream filters not applied → bitstream 변환 필요
Reference frame issue → 키프레임 문제
Audio underrun → 오디오 처리 못 따라감
Encoder frame queue is empty → 인코더 입력 부족
Dropping frame → 프레임 드롭
```

### 코덱 마이그레이션 - 단계별 진행

새 코덱 도입 시.

```
[단계별 마이그레이션]
1단계: 측정 (시청자 디바이스 분포)
   - 실제 시청자 코덱 지원율 측정
   - 디바이스별 통계

2단계: 시범 운영 (작은 트래픽)
   - 10% 시청자에 H.265 트랙 추가
   - 화질, 오류율 모니터링

3단계: 확대 (50% 시청자)
   - 안정성 확인
   - CDN 비용 절감 확인

4단계: 완전 도입
   - 모든 시청자에 H.265 제공
   - H.264는 폴백으로

5단계: H.264 단계적 종료
   - 시청자 디바이스 호환성 확인
   - 천천히 H.264 트랙 줄임
```

이 과정에 1년 이상 걸림.

### 면접 답변 - 디버깅 사례

"코덱 문제 어떻게 디버깅하셨어요?" 경험 질문.

```
"예시 들겠습니다.

상황: iOS 시청자만 영상이 안 보인다는 보고

진단:
1. 브라우저 개발자 도구로 video.error 확인
   → MEDIA_ERR_SRC_NOT_SUPPORTED

2. ffprobe로 영상 분석
   → H.265 영상, codec tag가 'hev1'

3. iOS Safari 호환성 확인
   → iOS는 hvc1만 지원, hev1 안 됨

해결:
- ffmpeg에 -tag:v hvc1 추가
- 재인코딩 후 iOS에서 정상 재생

학습:
- iOS 호환성 별도 트랙 필요
- 디바이스별 코덱 태그 차이 인지
- 모니터링에 디바이스별 실패율 추가
"
```

구체적 경험이 강한 답변.
