## 레벨 3-1 / 2강: NAL Unit과 비트스트림 구조

---

### H.264 비트스트림이 어떻게 구성되나

인코더가 만든 H.264 데이터가 바이트로 어떻게 흐르는지 본다.

H.264 비트스트림의 기본 단위는 **NAL Unit (Network Abstraction Layer Unit)**.

```
[H.264 비트스트림]
[NAL Unit 1][NAL Unit 2][NAL Unit 3]...

각 NAL Unit:
- 헤더 1바이트
- 페이로드 (RBSP - Raw Byte Sequence Payload)
```

---

### Network Abstraction Layer가 뭔가

이름의 의미를 보자.

```
H.264 구조:
- VCL (Video Coding Layer): 실제 영상 압축 데이터
- NAL (Network Abstraction Layer): 전송용 포장
```

VCL은 압축된 영상 자체. NAL은 그걸 어떻게 네트워크로 보낼지 추상화한 계층.

다양한 전송 방식에 맞춰 같은 영상을 다르게 포장 가능.

```
[같은 영상, 다른 컨테이너]
NAL Unit → MPEG-TS → SRT 전송
NAL Unit → RTP → WebRTC 전송
NAL Unit → fMP4 → HLS 전송
```

NAL이 있어서 영상 데이터와 전송 방식이 분리됨.

---

### NAL Unit 헤더

NAL Unit의 첫 1바이트.

```
0 1 2 3 4 5 6 7
+---------------+
|F|NRI|  Type   |
+---------------+

F (1비트): 금지 비트, 항상 0
NRI (2비트): 중요도 (Network Reference Indicator)
Type (5비트): NAL Unit 종류
```

**F (Forbidden bit)**: 0이어야 함. 1이면 손상된 NAL Unit.

**NRI**: 이 NAL이 얼마나 중요한지.
- 11: 매우 중요 (IDR, SPS, PPS)
- 10: 중요 (참조 프레임)
- 01: 덜 중요
- 00: 안 중요 (B-frame처럼 다른 게 참조 안 함)

손실 발생 시 NRI 보고 어떤 걸 우선 재전송할지 결정.

**Type**: NAL Unit이 어떤 데이터를 담고 있는지.

---

### NAL Unit Type 종류

주요 타입 (5비트, 0~31).

```
Type 1: 일반 슬라이스 (Non-IDR)
Type 2-4: 데이터 파티션 (안 씀)
Type 5: IDR 슬라이스 (Instantaneous Decoder Refresh)
Type 6: SEI (Supplemental Enhancement Information)
Type 7: SPS (Sequence Parameter Set)
Type 8: PPS (Picture Parameter Set)
Type 9: 액세스 유닛 구분자 (AUD)
Type 12: Filler 데이터
```

각 타입의 역할.

---

### SPS — Sequence Parameter Set

**SPS**: 영상 전체에 적용되는 파라미터.

```
SPS 내용:
- profile_idc: 프로파일 (Baseline/Main/High)
- level_idc: 레벨 (1080p60 = 4.0)
- pic_width_in_mbs: 가로 macroblock 수
- pic_height_in_map_units: 세로 macroblock 수
- bit_depth_luma: 휘도 비트 깊이 (보통 8)
- bit_depth_chroma: 색차 비트 깊이
- frame_mbs_only_flag: 프로그레시브 여부
- ...
```

영상 시작에 한 번 나옴. 디코더가 이걸 보고 디코딩 환경 설정.

세그먼트 단위 스트리밍(HLS)에서는 매 세그먼트 시작에 반복.

---

### PPS — Picture Parameter Set

**PPS**: 픽처(프레임) 단위 파라미터.

```
PPS 내용:
- pic_parameter_set_id: PPS 식별자
- entropy_coding_mode_flag: CAVLC/CABAC 선택
- num_slice_groups: 슬라이스 그룹 수
- num_ref_idx_l0_default_active: 참조 프레임 수
- weighted_pred_flag: 가중 예측 활성화
- pic_init_qp: 초기 QP
- chroma_qp_index_offset: 색차 QP 보정
- deblocking_filter_control_present: 디블로킹 필터
- ...
```

여러 PPS를 정의하고 슬라이스마다 어떤 PPS 쓸지 선택 가능.

---

### IDR Slice — 새 디코딩 시작점

**IDR (Instantaneous Decoder Refresh)**: 디코더 상태를 완전히 초기화하는 키프레임.

```
[IDR 발생 시]
디코더:
- 모든 참조 프레임 버림
- 새 GOP 시작
- 이전 영상 참조 안 함
```

일반 I-frame과 차이:
- 일반 I-frame: 다음 P-frame이 이전 GOP를 참조할 수 있음
- IDR I-frame: 이후 어떤 프레임도 IDR 이전을 참조 못 함

스트리밍 세그먼트 시작에 IDR 필수. 시청자가 중간부터 받아도 디코딩 가능.

---

### Slice — 프레임의 일부

한 프레임이 여러 슬라이스로 나뉠 수 있다.

```
[1080p 프레임을 4개 슬라이스로]
─────────────────
│   Slice 1     │
├───────────────┤
│   Slice 2     │
├───────────────┤
│   Slice 3     │
├───────────────┤
│   Slice 4     │
─────────────────
```

각 슬라이스 = 1 NAL Unit.

장점:
- 병렬 디코딩 가능 (CPU 코어별 슬라이스 처리)
- 오류 격리 (한 슬라이스 손상돼도 다른 슬라이스 정상)

---

### Annex B와 AVCC — 두 가지 비트스트림 포맷

NAL Unit을 어떻게 구분하는지 두 방식이 있다.

**Annex B**: Start Code로 NAL Unit 구분.

```
[Annex B 비트스트림]
00 00 00 01 [NAL Unit 1] 00 00 00 01 [NAL Unit 2] ...
└──────┬──────┘
   Start Code
```

3 또는 4바이트 0x00000001이 NAL Unit 시작 표시.

MPEG-TS, RTP 같은 스트리밍 환경에서 사용. 중간에 받아도 Start Code 찾으면 시작점 발견.

**AVCC**: 길이 prefix로 NAL Unit 구분.

```
[AVCC 비트스트림]
[4바이트 길이][NAL Unit 1][4바이트 길이][NAL Unit 2] ...
```

MP4, fMP4 컨테이너에서 사용. 파일 시스템 친화적.

---

### Annex B와 AVCC 변환

스트리밍 환경 바뀔 때 변환 필요.

```
[변환 예시]
Annex B (.ts 파일):
  00 00 00 01 67 64 00 28 ...  ← Start Code + SPS

AVCC (fMP4):
  00 00 00 07 67 64 00 28 ...  ← 길이(7) + SPS
```

hls.js의 트랜스먹싱이 .ts(Annex B) → fMP4(AVCC) 변환.

---

### Emulation Prevention Byte

문제 상황: NAL Unit 페이로드에 00 00 00 01이 우연히 나오면?

```
[원본 데이터]
... 12 34 00 00 00 01 56 78 ...
                ↑
       Start Code로 오인됨!
```

해결: **Emulation Prevention Byte**. 인코더가 00 00 01 패턴 발견하면 중간에 03 삽입.

```
[삽입 후]
... 12 34 00 00 03 00 01 56 78 ...
                   ↑
              03 삽입
```

디코더가 00 00 03 00 패턴 보면 03 제거.

작은 오버헤드지만 비트스트림 무결성 보장.

---

### Access Unit — 한 프레임의 모든 NAL Unit

**Access Unit (AU)**: 한 프레임을 구성하는 모든 NAL Unit의 묶음.

```
[Access Unit 예시 - IDR 프레임]
[AUD][SEI][SPS][PPS][IDR Slice 1][IDR Slice 2]...

AUD: Access Unit Delimiter (선택적)
SEI: 보조 정보
SPS, PPS: 파라미터 세트 (IDR 시 보통 포함)
Slice: 실제 영상 데이터
```

한 AU = 한 프레임. 비디오 PTS 기준.

---

### SEI — 보조 정보

**SEI (Supplemental Enhancement Information)**: 디코딩에 필수는 아니지만 유용한 정보.

```
SEI 종류:
- Picture Timing: 타임스탬프 정밀 표현
- Buffering Period: 버퍼 관리
- Closed Caption: 자막 (CEA-608/708)
- HDR Metadata: HDR10, Dolby Vision
- User Data: 임의 사용자 데이터
```

방송에서 자막을 SEI로 영상에 내장.

HDR 영상도 SEI로 색공간/밝기 정보 전달.

---

### Profile-Level-ID 분석

SDP나 SPS에서 보는 profile-level-id.

```
profile-level-id=42e01f

42 = profile_idc
e0 = constraint flags
1f = level_idc (10진수 31 = Level 3.1)
```

profile_idc 주요 값:
```
66  = Baseline Profile
77  = Main Profile
88  = Extended Profile
100 = High Profile
110 = High 10 Profile
122 = High 4:2:2 Profile
244 = High 4:4:4 Profile
```

constraint flags는 세부 제약사항.

### 비트스트림 분석 도구

H.264 비트스트림을 분석하는 도구.

**FFmpeg**:
```bash
# NAL Unit 정보 출력
ffmpeg -i video.h264 -loglevel debug

# 트랙 정보
ffprobe -v error -show_streams video.mp4
```

**h264bitstream**:
```bash
h264_analyze video.h264
# SPS, PPS, 슬라이스 정보 텍스트로
```

**Elecard StreamEye**:
GUI 도구. NAL Unit별 색상 표시. 매크로블록 분석.

---

### 2강 핵심 3줄

1. H.264 비트스트림은 NAL Unit 단위로 구성 — VCL(영상 데이터)과 NAL(전송 추상화) 분리, 헤더 1바이트(F+NRI+Type)
2. NAL Type별 역할 — SPS(영상 전체 설정), PPS(픽처 단위), IDR(완전 초기화 키프레임), SEI(보조 정보), Slice(실제 영상 데이터)
3. Annex B(00 00 00 01 Start Code, 스트리밍용)와 AVCC(길이 prefix, MP4용) 두 포맷 — Emulation Prevention Byte로 데이터 무결성 보장
