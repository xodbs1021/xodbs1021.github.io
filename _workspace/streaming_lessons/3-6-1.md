## 레벨 3-6 / 1강: 코덱 선택 의사결정 — 시나리오별 정답

---

### 코덱 결정의 6가지 요인

매번 만나는 질문 정리.

```
[결정 요인]
1. 시청자 디바이스 분포 (구형 vs 신형)
2. 콘텐츠 종류 (라이브 vs VOD)
3. 시청자 규모 (소규모 vs 대규모)
4. 비용 예산 (라이센스, 인코딩, CDN)
5. 지연 요구사항 (저지연 vs 호환성)
6. 화질 목표 (1080p vs 4K HDR)
```

이 6가지로 코덱 결정.

---

### 시나리오 1: 한국 라이브 게임 방송 (치지직)

```
[조건]
대규모 시청자: 피크 100만
일반 라이브: 게임 + 토크
모바일 시청자: 70%
저비용: 광고 모델
지연 OK: 18~25초

[결정]
영상: H.264 High Profile
오디오: AAC-LC 128 Kbps 48 kHz 스테레오
컨테이너: MPEG-TS (HLS) + fMP4 (LL-HLS)
프로토콜: RTMP → HLS

[이유]
호환성 100% (저가폰 포함)
라이센스 부담 없음
인코딩 인프라 검증
CDN 최적화 완료
```

### 시나리오 2: 글로벌 OTT (Netflix급)

```
[조건]
4K HDR 콘텐츠 중요
VOD 위주
다양한 디바이스
프리미엄 가입자
인코딩 시간 무관 (VOD)

[결정]
영상: H.265 Main 10 (4K HDR) + AV1 (모바일)
오디오: AAC + Dolby Atmos + E-AC-3
프로토콜: DASH + HLS (CMAF)
DRM: Widevine + FairPlay + PlayReady

[이유]
4K에서 H.265 절감 큼
AV1로 모바일 데이터 절약
다중 DRM 통합 필수
화질에 투자 가능
```

### 시나리오 3: WebRTC 화상 회의 (Zoom급)

```
[조건]
초저지연: 200ms 이하
양방향 통신
참가자 수십~수백
가변 네트워크 (3G~5G)

[결정]
영상: VP8 또는 H.264 Baseline + Simulcast
오디오: Opus 32~64 Kbps
프로토콜: WebRTC + SFU
처리: 에코 캔슬, 잡음 제거

[이유]
WebRTC 의무 코덱 (Opus)
Simulcast로 다인 통화 최적
H.264 Baseline은 저지연
SFU로 확장
```

### 시나리오 4: 모바일 라이브 송출

[조건]

[이유]
모바일 H.264 인코더 효율
배터리 절약

```

[이유]

[대역폭 비용 절감]

CMAF로 HLS+DASH 통합

```

### 호환성 우선순위

시청자 환경.

```
[모든 디바이스]
H.264 + AAC: 100% 호환

[옛 기기 제외]
H.265 + AAC: 약 70% 시청자

[최신 디바이스만]
AV1 + Opus: 30% 시청자

[디바이스 분포 모르면]
H.264 + AAC 안전
```

### 다중 코덱 운영 - 대형 플랫폼 전략

YouTube, Netflix 같은 곳.

```
[같은 콘텐츠를 여러 코덱으로]
1080p H.264 + AAC: 모든 시청자
1080p VP9 + Opus: Chrome/Android
4K H.265 + AAC: Apple/Smart TV
4K AV1 + Opus: 최신 기기

시청자 디바이스 자동 감지
가능한 가장 효율적인 트랙 제공
```

비용 늘지만:
- 시청자 경험 ↑
- 대역폭 ↓
- 새 디바이스 확장 ↑

### 결정 시 헷갈리는 부분

```
[Q: 라이브에 H.265 좋아 보이는데?]
A: 인코딩 비용 + 시청자 호환성 확인. 1080p 라이브엔 H.264로 충분.

[Q: VP9 안 쓰는 이유?]
A: AV1으로 흡수 중. 신규 도입 시 AV1 검토.

[Q: 4K가 필요한가?]
A: 시청자 디바이스/콘텐츠 확인. 모바일 위주면 1080p로 충분.

[Q: HDR 필수?]
A: 프리미엄 콘텐츠만. 일반 라이브엔 불필요.

[Q: Dolby Atmos는?]
A: 라이센스 + 비용 부담. Netflix급만.
```

### 면접 답변 - 종합

"코덱 선택 어떻게 하세요?" 종합 질문.

```
"6가지 요인을 고려합니다:
시청자 디바이스, 콘텐츠 종류, 시청자 규모, 비용, 지연, 화질.

저희 같은 한국 라이브 플랫폼:
- H.264 High Profile + AAC-LC 128 Kbps
- RTMP 수신 → HLS 배포
- 시청자 70% 모바일 + 저가폰 비율 큼
- 호환성 100% 우선

Netflix 같은 글로벌 OTT라면:
- 4K HDR에 H.265 또는 AV1
- 다중 코덱 트랙으로 디바이스 따라 자동 선택
- Dolby Atmos까지 프리미엄

WebRTC 회의 시스템이라면:
- 영상 VP8/H.264 Baseline
- 오디오 Opus (의무 코덱)
- SFU로 확장

핵심은 한 가지 코덱이 모든 경우에 답이 아니라는 것.
시나리오에 맞게 결정합니다."
```

### 1강 핵심 3줄

1. 코덱 결정은 6가지 요인 — 시청자 디바이스, 콘텐츠 종류, 규모, 비용, 지연, 화질
2. 한국 라이브는 H.264+AAC+RTMP→HLS, 글로벌 OTT는 H.265+AV1+AAC+Dolby Atmos, WebRTC는 VP8+Opus, PokeClip은 H.264+AAC+SRT+MPEG-TS
3. 대형 플랫폼은 다중 코덱 운영으로 호환성 + 효율 동시 — 한 가지 코덱이 모든 경우에 답 아님
