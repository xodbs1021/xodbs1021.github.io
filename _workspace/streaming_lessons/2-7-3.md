## 레벨 2-7 / 3강: WHIP/WHEP 실전 구축 — OBS부터 시청까지

---

### 전체 파이프라인 설계

WHIP/WHEP 기반 라이브 시스템 전체 구조.

```
[송출 측]
스트리머 OBS 30+
   │ WHIP (HTTP POST + WebRTC)
   ▼
[ingress 서버]
   - WHIP endpoint
   - 미디어 수신
   - 트랜스코딩 (선택)
   │
   ▼
[미디어 서버 (SFU)]
   - 시청자에게 분배
   - 화질별 분기
   │
   │ WHEP (HTTP POST + WebRTC)
   ▼
[시청자 브라우저]
   - WHEP 클라이언트
   - 영상 재생
```

각 단계에 어떤 도구를 쓸지 결정.

---

### 컴포넌트 선택

**Ingress 서버 (WHIP 받기)**:
```
선택지:
- Pion (Go): 가벼움, 직접 구현
- mediasoup (Node.js): 풍부한 기능
- Janus: C, 검증됨
- LiveKit: 통합 솔루션
- OvenMediaEngine: 한국 회사, 무료
```

**SFU (시청자 분배)**:
```
- LiveKit: 가장 쉬움
- Pion-SFU: Go 기반
- Mediasoup: 유연함
- Jitsi Videobridge: 검증됨
```

**시청자 측 (WHEP)**:
```
- 표준 브라우저 WebRTC API
- whep.js 라이브러리
- LiveKit Client SDK
```

이번 강의에선 가장 접근성 좋은 **OvenMediaEngine**으로 진행.

### OvenMediaEngine으로 WHIP 서버 구축

OvenMediaEngine은 WHIP과 WHEP 모두 네이티브 지원.

`Server.xml` 설정:

```xml
<Server version="8">
  <Name>OME-WHIP</Name>
  
  <Bind>
    <Providers>
      <!-- WHIP ingress -->
      <WebRTC>
        <Signalling>
          <Port>3333</Port>
          <TLSPort>3334</TLSPort>
        </Signalling>
        <IceCandidates>
          <IceCandidate>*:10000-10005/udp</IceCandidate>
        </IceCandidates>
      </WebRTC>
    </Providers>
    
    <Publishers>
      <!-- WHEP egress -->
      <WebRTC>
        <Signalling>
          <Port>3333</Port>
          <TLSPort>3334</TLSPort>
        </Signalling>
      </WebRTC>
    </Publishers>
  </Bind>
  
  <VirtualHosts>
    <VirtualHost>
      <Name>default</Name>
      
      <Applications>
        <Application>
          <Name>app</Name>
          <Type>live</Type>
          
          <Providers>
            <WebRTC>
              <CrossDomains>
                <Url>*</Url>
              </CrossDomains>
            </WebRTC>
          </Providers>
          
          <Publishers>
            <WebRTC>
              <Timeout>30000</Timeout>
              <CrossDomains>
                <Url>*</Url>
              </CrossDomains>
            </WebRTC>
          </Publishers>
          
          <!-- ABR ladder -->
          <OutputProfiles>
            <OutputProfile>
              <Name>passthrough</Name>
              <OutputStreamName>${OriginStreamName}</OutputStreamName>
              <Encodes>
                <Video>
                  <Bypass>true</Bypass>
                </Video>
                <Audio>
                  <Bypass>true</Bypass>
                </Audio>
              </Encodes>
            </OutputProfile>
          </OutputProfiles>
        </Application>
      </Applications>
    </VirtualHost>
  </VirtualHosts>
</Server>
```

실행:

```bash
docker run -d \
  --name ome \
  -p 3333:3333 \
  -p 3334:3334 \
  -p 10000-10005:10000-10005/udp \
  -v $(pwd)/Server.xml:/opt/ovenmediaengine/bin/origin_conf/Server.xml \
  airensoft/ovenmediaengine:latest
```

### OBS에서 WHIP 송출

OBS 30+ 설정:

```
[OBS 설정 - 방송 탭]
서비스: WHIP

서버: http://your-server-ip:3333/app/stream_name?direction=whip

Bearer Token: (있으면 입력, 없으면 빈칸)
```

OBS에서 송출 시작 누르면:
1. HTTP POST /app/stream_name?direction=whip
2. SDP Offer 전송
3. SDP Answer 받음
4. WebRTC 연결
5. 미디어 송출

### 시청자 측 WHEP 클라이언트

브라우저에서 WHEP으로 시청.

`watch.html`:

```html
<!DOCTYPE html>
<html>
<head>
  <title>WHEP Player</title>
</head>
<body>
  <video id="video" autoplay playsinline controls></video>
  
  <script>
    const SERVER_URL = 'http://your-server-ip:3333';
    const STREAM_NAME = 'stream_name';
    
    async function startWatching() {
      const video = document.getElementById('video');
      
      const pc = new RTCPeerConnection({
        iceServers: [{ urls: 'stun:stun.l.google.com:19302' }]
      });
      
      // 수신 전용
      pc.addTransceiver('audio', { direction: 'recvonly' });
      pc.addTransceiver('video', { direction: 'recvonly' });
      
      // 미디어 수신
      pc.ontrack = (event) => {
        if (event.track.kind === 'video' || event.track.kind === 'audio') {
          if (!video.srcObject) {
            video.srcObject = new MediaStream();
          }
          video.srcObject.addTrack(event.track);
        }
      };
      
      // Offer 생성
      const offer = await pc.createOffer();
      await pc.setLocalDescription(offer);
      
      // WHEP POST
      const whepUrl = `${SERVER_URL}/app/${STREAM_NAME}?direction=whep`;
      const response = await fetch(whepUrl, {
        method: 'POST',
        headers: { 'Content-Type': 'application/sdp' },
        body: offer.sdp,
      });
      
      if (!response.ok) {
        throw new Error(`WHEP error: ${response.status}`);
      }
      
      const answerSdp = await response.text();
      const resourceUrl = response.headers.get('Location');
      
      // Answer 적용
      await pc.setRemoteDescription({
        type: 'answer',
        sdp: answerSdp,
      });
      
      console.log('Connected to WHEP stream');
      
      // ICE 상태 모니터링
      pc.oniceconnectionstatechange = () => {
        console.log('ICE state:', pc.iceConnectionState);
      };
      
      // 종료 핸들러
      window.addEventListener('beforeunload', async () => {
        if (resourceUrl) {
          await fetch(resourceUrl, { method: 'DELETE' });
        }
        pc.close();
      });
    }
    
    startWatching().catch(console.error);
  </script>
</body>
</html>
```

이걸 웹 서버에 올리고 브라우저로 접속.

### 지연 측정

실제로 얼마나 빠른지 확인.

스트리머 측 OBS에 ms 단위 시계 띄움 (Time Display 플러그인).
시청자 측 화면 시계와 비교.

```
[측정 결과 예시]
WHIP/WHEP 직접 연결: 200~500ms
WHIP/WHEP + SFU 중간: 300~700ms
WHIP → HLS 변환: 5~10초
WHIP → LL-HLS 변환: 2~3초
```

지연을 줄이는 게 목적이면 WebRTC 끝까지 유지.

### 시청자 다중화 — 한 송출, 여러 시청 프로토콜

OvenMediaEngine은 한 송출을 여러 프로토콜로 동시에 분배.

```
[같은 스트림을 다양한 방식으로 시청]
시청자 A: WHEP (저지연)
시청자 B: LL-HLS (저지연 + iOS)
시청자 C: HLS (호환성)
시청자 D: RTMP (구버전 클라이언트)
```

각 시청자가 자기 환경에 맞게 선택.

스트리머는 한 번만 송출. 플랫폼이 자동으로 분기.

### 대규모 확장 — Edge Server

OvenMediaEngine의 Edge 모드.

```
[Origin-Edge 구조]
[OME Origin]
     │
     ├── [OME Edge 서울] → 한국 시청자
     ├── [OME Edge 도쿄] → 일본 시청자
     └── [OME Edge LA] → 미국 시청자
```

각 Edge가 Origin에서 스트림을 받아서 자기 지역 시청자에 분배.

Edge 설정:
```xml
<Provider>
  <Pull>
    <Url>https://origin.example.com</Url>
  </Pull>
</Provider>
```

지리적 분산으로 RTT 최소화.

### 인증 통합

플랫폼에 사용자 인증 통합.

1. 사용자가 로그인 → 백엔드에서 JWT 발급
2. 시청자 페이지에서 JWT를 WHEP에 포함

```javascript
// 시청자 측
const jwt = localStorage.getItem('auth_token');

const response = await fetch(whepUrl, {
  method: 'POST',
  headers: {
    'Content-Type': 'application/sdp',
    'Authorization': `Bearer ${jwt}`,
  },
  body: offer.sdp,
});
```

OvenMediaEngine은 SignedPolicy로 검증:

```xml
<SignedPolicy>
  <PolicyQueryKeyName>policy</PolicyQueryKeyName>
  <SignatureQueryKeyName>signature</SignatureQueryKeyName>
  <SecretKey>your_secret_key</SecretKey>
</SignedPolicy>
```

JWT 또는 Signed URL 토큰 검증.

### 실전 트러블슈팅

자주 발생하는 문제.

**문제 1: ICE 연결 실패**
원인: 방화벽 UDP 포트 차단.
해결: TURN 서버 설정 + TCP 폴백.

**문제 2: 영상은 오는데 검은 화면**
원인: 비디오 코덱 미지원 (예: H.265 시청자 미지원).
해결: H.264로 트랜스코딩.

**문제 3: 지연이 큼**
원인: SFU 거치는 단계 많음.
해결: 직접 Edge → 시청자 경로 최소화.

**문제 4: 시청자 끊기 후 자원 누적**
원인: DELETE 안 보내고 종료.
해결: Heartbeat 타임아웃으로 자동 정리.

### WHIP/WHEP 표준 vs 구현 차이

실제 구현마다 미묘한 차이.

```
[OvenMediaEngine]
URL 방식: ?direction=whip / ?direction=whep
경로: /app/stream_name

[LiveKit]
URL 방식: /rtc/ingress / /rtc/playback
인증: JWT 전용

[Cloudflare Stream]
URL 방식: /whip / /whep (표준)
ICE: 자체 TURN
```

OBS 같은 클라이언트는 표준 따르지만, 서버 측은 약간씩 다름. 호환성 확인 필요.
