### 라이브 트랜스코딩의 장애 빈도

24/7 운영하면서 마주치는 장애.

```
[흔한 장애 (월간 기준)]

NVENC 세션 누수: 2~5건
SRT 패킷 손실 폭증: 10~20건
스트리머 OBS 인코딩 변경: 50건+

CDN Origin Shield 장애: 0~1건

```

장애가 무서운 게 아니라 자동 복구 안 되는 게 무서움.

### 장애 처리의 원칙

1. **빨리 감지** - Mean Time To Detect (MTTD) 짧게
2. **자동 복구 우선** - 사람 호출 전에 시도
3. **격리** - 한 곳 장애가 다른 곳 안 퍼지게
4. **사후 분석** - 같은 장애 다시 안 나게

라이브는 분 단위 SLO. 사람 호출 후 대응하면 늦음.

### GPU Hang 자동 복구

NVIDIA 드라이버가 종종 hang. CUDA 호출 안 됨.

```python
class GPUHealthMonitor:
    def check_gpu(self, node):
        """GPU 동작 확인"""
        try:
            # nvidia-smi 호출 (5초 타임아웃)
            result = subprocess.run(
                ['nvidia-smi', '--query-gpu=utilization.gpu', '--format=csv,noheader'],
                timeout=5,
                capture_output=True
            )
            if result.returncode != 0:
                return False
            return True
        except subprocess.TimeoutExpired:
            return False
    
    def recover(self, node):
        """GPU 복구 시퀀스"""
        # 1단계: NVENC 세션 강제 정리
        self.kill_ffmpeg_processes(node)
        time.sleep(2)
        
        if self.check_gpu(node):
            return True
        
        # 2단계: 드라이버 리셋
        subprocess.run(['nvidia-smi', '--gpu-reset'])
        time.sleep(5)
        
        if self.check_gpu(node):
            return True
        
        # 3단계: 노드 재시작
        self.reboot_node(node)
        return False  # 재시작 후 복귀 대기
```

3단계 복구 시퀀스. 1단계로 80% 해결. 2단계로 95%. 3단계로 99%.

### FFmpeg 프로세스 좀비 감지

FFmpeg가 살아있지만 출력은 안 나오는 경우.

```python
class FFmpegHealthMonitor:
    def is_healthy(self, stream_id):
        """출력 파일 mtime 기반 헬스체크"""
        output_dir = f'/output/{stream_id}'
        if not os.path.exists(output_dir):
            return False
        
        files = os.listdir(output_dir)
        if not files:
            return False
        
        # 가장 최근 .m4s의 수정 시간
        latest = max(files, key=lambda f: os.path.getmtime(f'{output_dir}/{f}'))
        latest_mtime = os.path.getmtime(f'{output_dir}/{latest}')
        
        age = time.time() - latest_mtime
        return age < 10  # 10초 이내면 정상
    
    def recover(self, stream_id):
        # FFmpeg 프로세스 종료
        self.kill_ffmpeg(stream_id)
        
        # 새 프로세스 spawn
        self.start_ffmpeg(stream_id)
```

10초 간격 헬스체크. 좀비 발견 시 즉시 재시작. 시청자 5~15초 끊김.

### Region Failover

데이터센터 장애.

```python
class RegionFailover:
    def detect_failure(self, region):
        """region 장애 감지"""
        # 1. 트랜스코딩 노드 90% 이상 응답 없음
        unhealthy_ratio = self.get_unhealthy_node_ratio(region)
        if unhealthy_ratio > 0.9:
            return True
        
        # 2. 패키저 출력 완전 중단
        if not self.is_packager_outputting(region):
            return True
        
        return False
    
    def failover(self, from_region, to_region):
        """region 전환"""
        # 1. DNS 라우팅 변경
        self.update_dns(
            from_region.ingest_dns,
            to_region.ingest_endpoint
        )
        
        # 2. 활성 스트리머에게 알림
        for stream in self.get_active_streams(from_region):
            self.notify_reconnect(stream, to_region.ingest_endpoint)
        
        # 3. CDN 캐시 무효화
        self.cdn.purge_region(from_region)
```

DNS 전환은 즉각 안 됨 (TTL 캐시). 5~10분 끊김 예상.

Active-Active 구조면 즉시 fallback. 두 region 동시 운영.

### Active-Active 라이브

스트리머 한 명이 두 region에 동시 송출.

```
[Active-Active 구성]
스트리머 OBS → 서울 인제스트 (primary)
            → 도쿄 인제스트 (backup)
              ↓
        두 region 동시 트랜스코딩
              ↓
        패키저 출력 중 하나 선택
              ↓
        시청자
```

비용 2배. 핵심 스트리머만 적용.

장애 시 즉시 전환. 0초 끊김.

```python
class ActiveActiveSwitcher:
    def select_output(self, stream_id):
        """두 region 출력 중 건강한 것 선택"""
        seoul_health = self.check_region("seoul", stream_id)
        tokyo_health = self.check_region("tokyo", stream_id)
        
        if seoul_health and tokyo_health:
            return "seoul"  # 기본 우선
        elif tokyo_health:
            return "tokyo"
        elif seoul_health:
            return "seoul"
        else:
            return None  # 둘 다 죽음
```

### CDN 장애 처리

CDN 한 곳 장애 시 다른 CDN으로.

```python
class CDNSwitcher:
    def detect_cdn_failure(self):
        """CDN 헬스체크"""
        # 시청자 측 에러 비콘 모니터링
        error_rate = self.get_cdn_error_rate()
        if error_rate > 0.05:  # 5% 이상 에러
            return True
        
        # 응답 시간 폭증
        p99_latency = self.get_cdn_p99_latency()
        if p99_latency > 1000:  # 1초 이상
            return True
        
        return False
    
    def switch_cdn(self, from_cdn, to_cdn):
        """DNS 또는 클라이언트 라우터로 전환"""
        # GSLB (Global Server Load Balancing) 업데이트
        self.gslb.update_routing(
            from_cdn=from_cdn,
            to_cdn=to_cdn,
            weight=100  # 100% 전환
        )
```

Multi-CDN의 진짜 가치. 한 곳 죽어도 시청자 영향 최소.

### Chaos Engineering

일부러 장애 일으켜 시스템 검증.

```python
class ChaosMonkey:
    def random_kill_pod(self):
        """무작위로 K8s pod 종료"""
        if random.random() < 0.001:  # 0.1% 확률
            pod = random.choice(self.get_all_pods())
            kubectl.delete_pod(pod)
    
    def network_partition(self):
        """일시적 네트워크 단절"""
        if random.random() < 0.0001:
            target = random.choice(self.nodes)
            iptables.drop_packets(target, duration=10)
    
    def cpu_stress(self):
        """CPU 부하 인위적 증가"""
        stress(cpu_cores=4, duration=60)
```

운영 환경에서 매일 작은 장애 인위적 발생. 시스템이 자동 복구하는지 검증.

Netflix가 Chaos Monkey로 유명. 라이브 인프라엔 약간 부담스럽지만 비프로덕션에서는 표준.

### 알람 피로 - Alert Fatigue 방지

너무 많은 알람은 오히려 무시당함.

```
[알람 분류]
필요 즉시 행동: P0 (5분 내 호출)
필요 시간 내 행동: P1 (1시간 내)
정보 (대시보드): P2

[알람 빈도 제한]
같은 알람 5분 안에 중복 발송 안 함
한 시간 안에 같은 카테고리 5건 이상 발생 시 → 그룹 알람
```

P0 알람은 월 5건 이하 목표. 더 많으면 알람 룰 조정.

### 운영 자동화 - Runbook as Code

장애 대응을 코드로.

```python
# runbooks/gpu_hang.py
class GPUHangRunbook:
    description = "GPU 드라이버 hang 자동 복구"
    
    def check_trigger(self, alert):
        return alert.type == "gpu_unresponsive"
    
    def execute(self, alert):
        node = alert.affected_node
        
        # Step 1: GPU 리셋
        if self.reset_gpu(node):
            return "RECOVERED"
        
        # Step 2: 노드 재시작
        if self.reboot_node(node):
            return "RECOVERED"
        
        # Step 3: 노드 교체
        self.replace_node(node)
        return "RECOVERED_REPLACEMENT"
```

문서 대신 코드. 자동 실행 가능.
