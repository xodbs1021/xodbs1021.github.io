### FFmpeg로 SRT 송신

가장 흔한 사용 케이스부터. FFmpeg는 SRT를 잘 지원한다.

**기본 송신**:
```bash
ffmpeg -re -i input.mp4 \
  -c:v libx264 -preset veryfast -tune zerolatency \
  -c:a aac \
  -f mpegts \
  "srt://server.example.com:9000?mode=caller&latency=200"
```

핵심 옵션 분해:
- `-re`: 실시간 속도로 읽기 (파일을 라이브처럼)
- `-tune zerolatency`: 인코더 지연 최소화
- `-f mpegts`: MPEG-TS 컨테이너로 포장
- `latency=200`: SRT 버퍼 200ms

**FFmpeg로 SRT 수신**:
```bash
ffmpeg -i "srt://0.0.0.0:9000?mode=listener" \
  -c copy \
  output.ts
```

수신 후 파일 저장. `mode=listener`로 대기.

### SRT URL 옵션 전체

SRT URL의 쿼리 파라미터 모음.

```
[연결 모드]
mode=caller     # 연결 시작 (client)
mode=listener   # 연결 대기 (server)
mode=rendezvous # 양방향 연결

[Latency 관리]
latency=200          # 통합 latency (ms)
rcvlatency=200       # 수신 측 latency
peerlatency=200      # 송신 측이 보장하는 latency

[대역폭]
maxbw=10000000       # 최대 송신 (bps), 0=자동
inputbw=8000000      # 입력 대역폭 추정
oheadbw=25           # 오버헤드 비율 (%)

[버퍼]
sndbuf=12058624      # 송신 버퍼 (bytes)
rcvbuf=12058624      # 수신 버퍼

[암호화]
passphrase=secret    # 패스프레이즈
pbkeylen=16          # 키 길이 (16/24/32)

[Stream 식별]
streamid=#!::r=channel_a,u=token

[Live Mode 동작]
tlpktdrop=true       # 늦은 패킷 폐기
tsbpdmode=true       # Timestamp-based packet delivery

[기타]
transtype=live       # live or file
mss=1500             # MTU
fc=25600             # Flight Flag Size
```

대부분 기본값으로 충분. latency만 환경에 맞게 조정.

### OBS에서 SRT 송신 설정

OBS 25 이상에서 SRT 지원. 설정 방법.

```
[OBS 설정 - 방송 탭]
서비스: Custom
서버: srt://server.example.com:9000?mode=caller&latency=200
스트림 키: (빈 값 또는 추가 인증)
```

서버 URL에 모든 옵션 포함. 별도 입력 칸 없음.

**OBS의 SRT 한계**:
- streamid 옵션 지원 제한적
- bonding 미지원
- 통계 표시 부족

전문가용은 OBS보다 FFmpeg나 전용 SRT 송출기(Haivision Makito, MediaSputnik) 사용.

### SRT 서버 구축 — SRS

가장 인기 있는 오픈소스 SRT 서버.

**SRS (Simple Realtime Server)**:
중국 회사가 만든 오픈소스. RTMP, HLS, SRT, WebRTC 모두 지원.

```bash
# Docker로 SRS 실행
docker run --rm -it \
  -p 1935:1935 \
  -p 1985:1985 \
  -p 8080:8080 \
  -p 10080:10080/udp \
  ossrs/srs:5
```

설정 파일 `srs.conf`:

```
listen              1935;
max_connections     1000;
srt_server {
    enabled         on;
    listen          10080;
    maxbw           1000000000;
    connect_timeout 4000;
    peer_idle_timeout 10000;
}

vhost __defaultVhost__ {
    srt {
        enabled     on;
        srt_to_rtmp on;     # SRT → RTMP 변환
    }
    rtmp_to_rtc on;          # RTMP → WebRTC 변환
    rtc_to_rtmp on;
}
```

**송신 (스트리머)**:
```bash
ffmpeg -re -i input.mp4 \
  -c copy -f mpegts \
  "srt://srs-server:10080?streamid=#!::r=live/test"
```

**시청 (HLS로)**:
```
http://srs-server:8080/live/test.m3u8
```

SRT로 받아서 자동으로 HLS로 변환. 시청자는 일반 HLS로 시청.

### SRT를 RTMP로 변환

SRT로 받아서 RTMP 플랫폼(치지직 등)으로 보낼 때.

**nginx-rtmp 보조**:
nginx-rtmp는 SRT 미지원. SRS 또는 FFmpeg 중계 사용.

```bash
# FFmpeg로 SRT 받아서 RTMP로 전달
ffmpeg \
  -i "srt://0.0.0.0:9000?mode=listener" \
  -c copy -f flv \
  "rtmp://chzzk-server/live/stream_key"
```

지연이 추가되지만 (몇 백 ms), SRT의 안정성 확보.

### 모바일 방송 SDK

iOS/Android에서 SRT로 송출하는 SDK.

**Larix Broadcaster**:
무료 모바일 SDK. SRT, RTMP 모두 지원.

```
[Larix 설정]
연결 URL: srt://server:9000?streamid=mychannel
Latency: 500ms
Encryption: AES-128 + passphrase
```

**SRT iOS SDK**:
Haivision 공식 iOS SDK. 유료.

**libsrt 직접 사용**:
크로스 플랫폼 C++ 라이브러리. 직접 통합.

```swift
// iOS Swift 예시 - SRT 송신
import SRT

let socket = SRTSocket()
socket.connect(host: "server.example.com", port: 9000)
socket.setOption(.latency, value: 500)

while streaming {
  let frame = captureCameraFrame()
  let encoded = encodeH264(frame)
  let tsPacket = wrapInMPEGTS(encoded)
  socket.send(tsPacket)
}
```

### SRT 연결 디버깅

문제 발생 시 확인할 것들.

**1. 패킷 손실률**
```
pktRcvLoss / pktRecv > 5% → 네트워크 품질 문제
```

**2. 재전송률**
```
pktRetrans / pktSent > 10% → latency 늘리거나 inputbw 줄이기
```

**3. Dropped 패킷**
```
pktRcvDrop > 0 → latency 부족. 늘려야 함
```

**4. RTT 변동**
```
RTT가 크게 흔들리면 네트워크 jitter 큼 → latency 4×RTT 권장
```

**FFmpeg 디버깅 모드**:
```bash
ffmpeg -loglevel debug -i "srt://..." -c copy output.ts
```

자세한 SRT 로그 출력.

### PokeClip에서 SRT 사용 패턴

PokeClip 같은 분석 파이프라인에서.

```
[스트리머 OBS]
   │ RTMP → 치지직 (방송)
   │
   └── SRT+MPEG-TS → PokeClip EC2 (분석용)
                        │ 4개 오디오 트랙
                        ▼
                     FFmpeg demux
                        │
                        ├── 게임 소리 → 분석
                        ├── 마이크 → STT
                        ├── 디스코드 → 분석
                        └── 후원 → 감지
```

SRT 설정:
```
srt://pokeclip-server:9000?mode=caller&latency=500&passphrase=secret&streamid=streamer123
```

서버 측:
```python
# 받은 SRT 스트림을 FFmpeg에 넘김
import subprocess

cmd = [
    'ffmpeg',
    '-i', f'srt://0.0.0.0:9000?mode=listener&streamid={streamer_id}',
    '-map', '0:a:0', '-c:a', 'copy', f'/tmp/{streamer_id}/game.aac',
    '-map', '0:a:1', '-c:a', 'copy', f'/tmp/{streamer_id}/mic.aac',
    '-map', '0:a:2', '-c:a', 'copy', f'/tmp/{streamer_id}/discord.aac',
    '-map', '0:a:3', '-c:a', 'copy', f'/tmp/{streamer_id}/donation.aac',
]
subprocess.Popen(cmd)
```
