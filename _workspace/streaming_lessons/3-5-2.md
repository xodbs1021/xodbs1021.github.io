## 레벨 3-5 / 2강: AV1의 등장 — H.265를 건너뛰는 산업 동맹

---

### AV1 이전의 상황 정리

2015년 즈음.

```
[코덱 전선]
H.265 진영: Apple, Netflix, Disney, 방송사
VP9 진영: Google, Mozilla, Cisco

H.265 문제: 라이센스 비용 + 복잡한 풀
VP9 문제: 보급률 낮음
```

각 진영이 따로 가서 분열. 모두에게 비효율.

이런 상황에서 새 동맹.

---

### AOMedia의 탄생 — 2015년

**AOMedia (Alliance for Open Media)**: H.265 라이센스 부담을 함께 피하려는 거대 기업 연합.

```
[AOMedia 창립 멤버]
Google
Amazon
Cisco
Intel
Microsoft
Mozilla
Netflix
NVIDIA (이후 합류)
Apple (2018년 합류!) ← 큰 사건
```

H.265 진영이었던 **Netflix와 Apple까지 합류**.

이게 결정적. AV1이 진짜 표준이 될 가능성 생김.

---

### AV1의 발표 — 2018년

3년 개발 후 표준 1.0 완성.

```
[AV1 (AOMedia Video 1) 목표]
H.265 대비 30% 비트레이트 절감
로열티 무료
오픈소스 구현
```

기존 기술 통합:
- Google의 VP10 (취소)
- Mozilla의 Daala
- Cisco의 Thor

이걸 합쳐서 AV1.

---

### AV1 vs H.265 vs VP9 — 압축률

```
[같은 화질 기준 비트레이트]
H.264: 100% (기준)
VP9: 60~70%
H.265: 50~60%
AV1: 40~50%
```

AV1이 가장 효율적.

```
[4K 영상 시청자 1명당 데이터]
H.264: 25 Mbps
H.265: 12 Mbps
VP9: 14 Mbps
AV1: 10 Mbps
```

대규모 OTT에 매년 수십억 달러 절감 가능.

---

### AV1의 엄청난 부담 — 인코딩 시간

압축률은 좋은데 인코딩이 극도로 느리다.

```
[같은 화질 인코딩 시간]
H.264 medium: 1x
H.265 medium: 약 10x 느림
AV1 libaom: 100~500x 느림!!
```

이게 보급의 가장 큰 장벽.

```
[1시간 영상 인코딩 시간 - 1080p]
H.264 x264 medium: 30분
H.265 x265 medium: 5시간
AV1 libaom: 50~250시간
```

실시간 인코딩 거의 불가능했음.

---

### SVT-AV1 — Intel + Netflix의 해결

2019년 Intel과 Netflix가 빠른 AV1 인코더 발표.

```
[SVT-AV1]
- libaom 대비 5~10배 빠름
- 화질은 약간 떨어짐 (3~5%)
- 실시간 1080p AV1 인코딩 가능 (고성능 서버)
- 오픈소스
```

이게 AV1 보급의 결정타.

```bash
# SVT-AV1 인코딩 (실용적)
ffmpeg -i input.mp4 \
  -c:v libsvtav1 \
  -preset 7 \
  -crf 35 \
  output.mp4

# libaom (느리지만 화질 최고)
ffmpeg -i input.mp4 \
  -c:v libaom-av1 \
  -crf 30 \
  -b:v 0 \
  output.mp4
```

대부분 SVT-AV1 사용.

### 하드웨어 인코더 보급

2022년부터 GPU에 AV1 인코더 내장.

```
[AV1 하드웨어 인코더]
NVIDIA Ada Lovelace (RTX 4000): 2022년
Intel Arc GPU: 2022년
Intel 13세대 CPU+ (Quick Sync): 2022년
AMD RDNA 3+ (RX 7000): 2022년
```

GPU 인코더로 실시간 4K AV1 가능.

```bash
# NVIDIA AV1
ffmpeg -i input.mp4 \
  -c:v av1_nvenc \
  -preset p4 \
  -b:v 2000k \
  output.mp4

# Intel AV1
ffmpeg -i input.mp4 \
  -c:v av1_qsv \
  -b:v 2000k \
  output.mp4
```

### 디코딩 부담

다행히 디코딩은 인코딩만큼 무겁지 않음.

```
[디코딩 CPU 부하 - 1080p60]
H.264: 매우 낮음
H.265: 보통
AV1: 보통 (H.265보다 약간 더)
VP9: 보통
```

근데 하드웨어 디코더 보급이 늦음.

```
[AV1 하드웨어 디코더 보급]
스마트TV: 2021년 이후 최신 모델
iPhone: 15 Pro부터 (2023년)
Android: Galaxy S22+ (2022년)
PC GPU: RTX 3000+ / RX 6000+
구형 기기: 디코딩 불가
```

저가폰은 여전히 AV1 못 본다.

### AVIF — AV1의 이미지 포맷

AV1 영상 기술을 이미지에 적용.

```
[AVIF]
- AV1 키프레임을 이미지로 사용
- JPEG 대비 50% 작음
- HDR 지원
- 무손실/손실 모두 가능
```

웹 이미지 표준 자리 차지하는 중.

```bash
# AVIF 인코딩
ffmpeg -i input.jpg -c:v libaom-av1 -still-picture 1 -crf 30 output.avif

# 또는 전용 도구
avifenc input.jpg output.avif
```

브라우저 지원:
- Chrome 85+
- Firefox 93+
- Safari 16+

JPEG 대안. 점진적 확산.

### AV1 인코딩 실전 — 가이드

용도별 명령.

```bash
# VOD 최고 화질 (시간 무관)
ffmpeg -i input.mp4 \
  -c:v libsvtav1 \
  -preset 4 \
  -crf 30 \
  -g 240 \
  -pix_fmt yuv420p10le \
  -c:a libopus -b:a 128k \
  output.mp4

# VOD 균형
ffmpeg -i input.mp4 \
  -c:v libsvtav1 \
  -preset 8 \
  -crf 35 \
  -c:a libopus -b:a 128k \
  output.mp4

# 라이브 (NVENC 필요)
ffmpeg -i input \
  -c:v av1_nvenc \
  -preset p4 \
  -tune ll \
  -rc cbr \
  -b:v 2000k \
  -g 60 \
  -c:a aac -b:a 128k \
  -f flv rtmp://...
```

`preset 8`이 균형. `preset 4`가 화질 우선. `preset 13`이 속도 우선.

### AV1 코덱 문자열

MIME 표기.

```
av01.0.05M.08

av01 = AV1
0 = Profile (Main)
05M = Level (4.0 Main)
08 = bit depth (8-bit)
```

브라우저 지원 확인:
```javascript
MediaSource.isTypeSupported('video/mp4; codecs="av01.0.05M.08"')
```

### 코덱 종합 비교

```
[2024년 코덱 비교 종합]

호환성:
H.264 > H.265 > VP9 > AV1

압축률:
AV1 > H.265 > VP9 > H.264

인코딩 속도:
H.264 > H.265 > VP9 > AV1

라이센스 비용:
H.264 > H.265 (>> 비싸짐)
VP9, AV1: 무료

하드웨어 지원:
H.264 ≈ H.265 > AV1 > VP9
```

각 코덱의 강점이 다름.

```
[AV1 도입 OK]

```

치지직 = AV1 불필요. 향후 5년도 H.264 유지가 정답.

YouTube, Netflix = AV1 도입 진행 중.

### 미래의 코덱 경쟁

AV1 이후.

**VVC (H.266)**:
```
[VVC]
- 2020년 표준화 (MPEG)
- AV1 대비 약간 우수
- 라이센스 다시 복잡
- 도입 더딘 중
```

**EVC (MPEG-5 EVC)**:
```
- Baseline Profile: 무료
- Main Profile: 유료
- AV1과 경쟁 중
```

**LCEVC**:
```
- 기존 코덱에 화질 레이어 추가
- 호환성 유지하며 효율 향상
```

당분간 H.264 + H.265 + AV1 + VP9가 공존.

### 2강 핵심 3줄

1. AV1은 2018년 AOMedia (Google + Apple + Netflix + Microsoft + Mozilla 등) 합작 무료 표준 — H.265 대비 30%, H.264 대비 50% 대역폭 절감
2. 인코딩 100~500배 느림 → SVT-AV1 (Intel+Netflix)로 해소, NVIDIA RTX 4000+ / Intel Arc / AMD RDNA 3+가 하드웨어 인코딩 지원
3. YouTube/Netflix/Twitch가 점진 도입 중, 시청자 측 보급 5~10년 예상 — 한국 라이브는 H.264 유지가 정답, AVIF가 JPEG 대체 시작
