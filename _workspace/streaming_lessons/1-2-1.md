## 레벨 1-2 / 1강: 해상도와 FPS — 화질을 수치로 정의하는 방법

---

### 해상도 — 픽셀이 몇 개냐

해상도는 화면을 구성하는 픽셀 수다.

```
1920 × 1080 = 2,073,600 픽셀 (약 200만)
3840 × 2160 = 8,294,400 픽셀 (약 830만) → 4K
```

근데 같은 1920×1080이라도 화면 크기에 따라 체감 화질이 다르다.

27인치 모니터의 1080p와 6인치 스마트폰의 1080p — 폰이 훨씬 선명해 보인다.

같은 픽셀 수를 더 작은 면적에 넣으면 **PPI(Pixel Per Inch)** 가 높아진다.

```
27인치 1080p: 82 PPI
6인치 1080p: 367 PPI   ← 폰이 4배 더 촘촘
```

스트리밍에서 이게 왜 중요하냐면, 모바일로 보는 시청자에게 1080p를 억지로 줄 필요가 없다는 뜻이다. 480p로도 충분히 선명하게 보인다. **ABR에서 모바일 화질 사다리를 낮게 잡는 이유**다.

---

### FPS — 초당 몇 장이냐

FPS(Frames Per Second): 1초에 보여주는 이미지 수.

```
24fps: 영화 (자연스러운 모션 블러)
30fps: 일반 방송, 유튜브 기본
60fps: 게임 방송, 스포츠
120fps: 고사양 게임, 스포츠 중계 (아직 보급 중)
```

치지직 스트리머들이 대부분 **1080p60**으로 방송하는 이유가 있다. 게임 방송이라서다.

30fps vs 60fps 차이가 가장 크게 체감되는 게 **빠른 움직임**이다.

```
30fps: 총 발사 → 이전 프레임에서 다음 프레임까지 33ms 공백
       빠른 움직임이 뭉개져 보임 (모션 블러)

60fps: 총 발사 → 16ms 공백
       움직임이 훨씬 선명하게 보임
```

FPS를 올리면 데이터량이 어떻게 되나?

```
1080p30: 프레임 30장/초
1080p60: 프레임 60장/초 → 데이터 2배?
```

실제로는 2배가 아니다. **1.5배 정도**. 이유가 있다.

60fps에서는 연속 프레임 간 차이가 작다. 30fps는 33ms 간격이고 60fps는 16ms 간격이다. 16ms 간격의 P-frame은 33ms 간격보다 변화량이 작아서 더 잘 압축된다.

---

### 해상도와 FPS가 비트레이트에 미치는 영향

1강에서 배운 것처럼, 실제 전송되는 데이터량은 압축 후 기준이다.

치지직 같은 플랫폼의 권장 설정:

```
1080p60: 6,000 Kbps (6 Mbps)
1080p30: 4,500 Kbps (4.5 Mbps)
720p60:  4,500 Kbps
720p30:  3,000 Kbps
480p30:  1,500 Kbps
360p30:  800 Kbps
```

해상도가 2배 올라가도 비트레이트가 딱 2배가 되지 않는다. 픽셀 수가 늘어도 압축률이 함께 좋아지기 때문이다.

---

### 1080i vs 1080p — 아직도 가끔 나오는 개념

방송 장비에서 가끔 나오는 표기법:

**1080p (Progressive)**: 매 프레임 전체를 그린다. 우리가 일반적으로 쓰는 방식.

**1080i (Interlaced)**: 홀수 라인과 짝수 라인을 번갈아 그린다. 옛날 CRT TV를 위한 방식.

```
1080p: 프레임 1에 전체 1080줄 그림
1080i: 프레임 1에 홀수 줄만(540줄) → 프레임 2에 짝수 줄만(540줄)
```

왜 이렇게 했냐면 1080i는 실질적으로 540p를 60fps로 보내는 것과 같다. 대역폭 절약. CRT 특성상 이렇게 해도 자연스럽게 보였다.

지금은 LCD/OLED라 1080i를 쓸 이유가 없다. 근데 방송 장비에서 아직 나온다.

FFmpeg에서 `-vf yadif` 옵션이 바로 인터레이스를 프로그레시브로 변환하는 디인터레이스 필터다.

### 1강 핵심 3줄

1. 해상도는 픽셀 수 — 같은 해상도라도 화면 크기에 따라 PPI가 달라서 모바일엔 낮은 화질도 충분히 선명하다
2. FPS는 모션 부드러움 — 게임 방송이 60fps인 이유, 30fps→60fps는 데이터 2배가 아니라 1.5배
3. 1080i는 옛날 CRT용 방식, 지금은 1080p가 표준 — FFmpeg `-vf yadif`로 변환 가능
