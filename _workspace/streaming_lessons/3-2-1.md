## 레벨 3-2 / 1강: AAC가 등장한 배경 — MP3로는 부족했다

---

### 1990년대 초 — MP3의 탄생

오디오 압축의 시작점.

```
[MP3 등장]
1989년: Fraunhofer가 MPEG-1 Audio Layer 3 발표
1993년: ISO 표준화
1990년대 중반: 인터넷 음악 혁명
```

MP3 (MPEG-1 Audio Layer 3)는 인터넷 음악 시대를 만들었다.

CD 음질 (1.4 Mbps)을 128 Kbps로 압축. 약 11배.

Napster, iPod, 인터넷 라디오 모두 MP3 위에서 가능했다.

---

### MP3의 한계가 드러나다

성공적이었지만 시간이 지나면서 한계가 보였다.

**1. 다중 채널 미지원**
MP3는 본질적으로 스테레오까지. 5.1 채널 같은 서라운드 표현 불가.

```
[MP3 채널 한계]
모노: ✅
스테레오: ✅
5.1 채널: ❌ (MP3 표준 외 확장 필요)
7.1 채널: ❌
```

DVD 등장 (1995) 후 5.1 채널 표준 필요. MP3로 안 됨.

**2. 저비트레이트 음질 나쁨**
64 Kbps 이하에서 MP3는 매우 거친 소리.

```
[MP3 비트레이트별 음질]
320 Kbps: 매우 좋음 (CD에 가까움)
192 Kbps: 좋음
128 Kbps: 보통 (대부분 사람 만족)
96 Kbps: 떨어짐
64 Kbps: 나쁨 (음악)
32 Kbps: 음성도 거침
```

모바일 환경(낮은 대역폭) 등장 후 저비트레이트 코덱 필요.

**3. 알고리즘의 한계**
MP3는 폴리페이즈 필터 + MDCT 일부 사용. 더 효율적인 압축 가능한 알고리즘이 발견됨.

**4. 라이센스 비용**
1990년대~2010년대 MP3 사용에 Fraunhofer 라이센스 비용 필요.

---

### MPEG의 후속 작업 — AAC 개발

ISO/IEC MPEG 그룹이 MP3의 한계 해결을 위해 시작.

```
[AAC 개발 일정]
1994년: 연구 시작
1997년: MPEG-2 Part 7로 표준화 (ISO/IEC 13818-7)
1999년: MPEG-4 Part 3에 포함 (ISO/IEC 14496-3)
2003년: HE-AAC 추가 (저비트레이트용)
2006년: HE-AAC v2 추가
```

여러 회사 공동 개발:
- Fraunhofer IIS (독일, MP3 만든 곳)
- AT&T
- Sony
- Nokia
- Dolby

---

### AAC의 정식 명칭

여러 이름으로 불린다.

```
[AAC 명칭]
- AAC: Advanced Audio Coding
- MPEG-2 Part 7
- MPEG-4 Part 3 (오디오 부분)
- ISO/IEC 13818-7 (MPEG-2)
- ISO/IEC 14496-3 (MPEG-4)
```

같은 코덱의 다양한 이름.

MPEG-2 버전과 MPEG-4 버전이 약간 다름:
- MPEG-2 AAC: 옛 버전
- MPEG-4 AAC: 현재 표준 (Object Type 명시)

---

### AAC가 MP3보다 좋은 이유

기술적 개선점들.

**1. MDCT 전체 사용**
MP3는 폴리페이즈 + MDCT 부분 사용. AAC는 순수 MDCT.

```
[MDCT 윈도우]
MP3: 36 또는 18 샘플 (32 서브밴드 + MDCT)
AAC: 2048 또는 256 샘플 (직접 MDCT)
```

긴 윈도우 = 주파수 분해능 향상.

**2. 더 정교한 사이코어쿠스틱 모델**
사람 청각 모델링이 더 정확.

**3. Temporal Noise Shaping (TNS)**
빠른 변화(transient) 처리 개선.

```
[Transient 처리 - 드럼 소리 예시]
MP3: 약간의 pre-echo 발생 (드럼 치기 전에 소음)
AAC: TNS로 pre-echo 제거
```

**4. Prediction (예측)**
연속 프레임 간 예측으로 압축률 향상.

**5. Perceptual Noise Substitution (PNS)**
랜덤 노이즈는 따로 표현 안 하고 메타데이터로.

**6. 더 효율적 양자화**
주파수 영역에서 정밀한 비트 분배.

---

### MP3 vs AAC 비교 — 종합

```
[같은 비트레이트에서 음질]

64 Kbps 스테레오 음악:
  MP3: 매우 나쁨
  AAC: 보통 (들을만함)

96 Kbps:
  MP3: 나쁨
  AAC: 좋음

128 Kbps:
  MP3: 보통 (대부분 만족)
  AAC: 매우 좋음

192 Kbps:
  MP3: 좋음
  AAC: 거의 CD 음질

256 Kbps:
  MP3: 매우 좋음
  AAC: 무손실에 가까움
```

비트레이트가 낮을수록 AAC 차이가 큼.

AAC 128 Kbps ≈ MP3 192 Kbps 음질.

---

### Apple의 채택 — iPod와 iTunes

AAC 보급의 결정적 계기.

```
[Apple의 AAC 채택]
2003년: iTunes Store 출시
- 음원 판매를 AAC로
- MP3 안 씀 (라이센스 + 음질 + DRM 통합)

2005년: iPod 시리즈 AAC 표준화
2007년: iPhone 출시
- 음악 + 동영상 모두 AAC
```

Apple 생태계가 AAC를 사실상 표준으로 만듦.

이전엔 MP3가 압도적이었지만, 모바일 시대로 가면서 AAC 점유율 폭증.

---

### iTunes의 음원 포맷

당시 iTunes Store 음원 사양.

```
[iTunes Store AAC]
컨테이너: M4A (MP4 기반)
코덱: AAC-LC
비트레이트: 128 Kbps (초기) → 256 Kbps (2009년부터)
샘플링: 44.1 kHz
채널: 스테레오
DRM: FairPlay (2003~2009), 이후 없음
```

iPhone, iPad, Mac에서 모두 재생.

256 Kbps AAC는 사실상 무손실급. 대부분의 사람이 원본 CD와 구분 못 함.

---

### YouTube의 AAC 채택

영상 플랫폼도 AAC 표준.

```
[YouTube 오디오 포맷]
2005년 출범: MP3 + 폐쇄 코덱
2010년대: AAC 표준
현재: AAC + Opus (저비트레이트)
```

YouTube 영상의 오디오:
- 일반 화질: AAC 128 Kbps
- 고화질: AAC 256 Kbps
- 음악: Opus 또는 AAC 256 Kbps

---

### HLS의 AAC 의무화

Apple이 만든 HLS는 AAC 필수.

```
[HLS 오디오 코덱]
필수: AAC (HE-AAC 포함)
선택: AC-3, EC-3
미지원: MP3, Opus (옛 버전)
```

iPhone이 HLS의 시작이니 당연히 AAC 강제.

라이브 스트리밍에서 AAC가 표준이 된 핵심 이유.

---

### 방송 표준 — AAC의 영역 확장

디지털 방송도 AAC 채택.

```
[방송 AAC 채택]
DVB (유럽): AAC 표준
ISDB-T (일본/한국 일부): AAC 표준
ATSC 3.0 (미국 차세대): AAC
한국 디지털 방송: AAC

비교:
- ATSC 1.0 (옛 미국): AC-3
- 옛 유럽: MP2 (MPEG Layer 2)
```

지상파, 위성, 케이블 디지털 방송 대부분 AAC.

---

### 1강 핵심 3줄

1. AAC는 MP3의 다중 채널 미지원, 저비트레이트 음질 문제, 알고리즘 한계를 해결하기 위해 1997년 표준화 — Fraunhofer + AT&T + Sony + Nokia + Dolby 공동 개발
2. MDCT 전체 사용, TNS로 transient 처리, Prediction 등으로 같은 비트레이트에 MP3보다 우수 — AAC 128 Kbps ≈ MP3 192 Kbps 음질
3. iTunes (2003), iPod, iPhone, YouTube, HLS, 디지털 방송이 AAC 채택하면서 사실상 오디오 표준 — 라이브 스트리밍의 기본 코덱
