## 레벨 4-2 / 3강: ABR Ladder 실전 구성 — FFmpeg과 Shaka Packager

### 가장 단순한 사다리 — FFmpeg 한 명령

3단계 사다리를 한 명령으로.

```bash
ffmpeg -i input.mp4 \
  -filter_complex "\
    [0:v]split=3[v1080][v720_in][v480_in]; \
    [v720_in]scale=1280:720[v720]; \
    [v480_in]scale=854:480[v480]" \
  \
  -map "[v1080]" -map 0:a \
    -c:v libx264 -preset veryfast -tune zerolatency \
    -b:v 6000k -maxrate 6000k -bufsize 12000k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 128k -ar 48000 \
  \
  -map "[v720]" -map 0:a \
    -c:v libx264 -preset veryfast -tune zerolatency \
    -b:v 3000k -maxrate 3000k -bufsize 6000k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 128k -ar 48000 \
  \
  -map "[v480]" -map 0:a \
    -c:v libx264 -preset veryfast -tune zerolatency \
    -b:v 1500k -maxrate 1500k -bufsize 3000k \
    -g 120 -keyint_min 120 -sc_threshold 0 \
    -c:a aac -b:a 96k -ar 48000 \
  \
  -f hls \
  -hls_time 6 \
  -hls_segment_type fmp4 \
  -hls_flags delete_segments+independent_segments+program_date_time \
  -hls_segment_filename 'stream_%v/seg_%05d.m4s' \
  -hls_fmp4_init_filename 'init.mp4' \
  -master_pl_name master.m3u8 \
  -var_stream_map "v:0,a:0,name:1080p v:1,a:1,name:720p v:2,a:2,name:480p" \
  \
  'stream_%v/playlist.m3u8'
```

이게 라이브 인제스트 + 트랜스코딩 + 패키징 통합.

### GOP 정렬이 핵심

ABR 사다리에서 가장 중요한 부분.

**모든 화질이 같은 시점에 키프레임을 박아야 한다.**

```
[잘못된 경우]
1080p: I-P-P-P-I-P-P-P-...  (6초마다 I)
720p:  I-P-P-I-P-P-I-P-...   (4.5초마다 I)
480p:  I-P-P-P-P-I-P-P-...   (7.5초마다 I)

→ 세그먼트 경계가 안 맞음
→ ABR 전환 시 영상 끊김
```

```
[올바른 경우]
1080p: I-P-P-P-I-P-P-P-...  (6초마다 I)
720p:  I-P-P-P-I-P-P-P-...  (6초마다 I)
480p:  I-P-P-P-I-P-P-P-...  (6초마다 I)

→ 세그먼트 경계 정확히 맞음
→ ABR 전환 시 끊김 없음
```

이걸 보장하는 옵션이 모든 화질에 동일하게:

```bash
-g 120                # GOP 120프레임 (2초@60fps, 4초@30fps)
-keyint_min 120       # 최소 간격도 같게
-sc_threshold 0       # 씬 체인지 키프레임 비활성
```

`-sc_threshold 0`이 결정적. 씬 전환 감지로 자동 키프레임 박지 못하게 막음.

### 진짜 CBR 강제

ABR에서 BANDWIDTH 값이 peak 기준이라 진짜 CBR이 필요.

```bash
# 진짜 CBR (peak = average)
-b:v 6000k \
-maxrate 6000k \
-minrate 6000k \      # x264 NAL filler로 빈 비트 채움
-bufsize 6000k        # 작게 잡아 변동 억제
```

`-minrate` 추가하면 비트가 빈 시점에 패딩으로 채운다. 트래픽 낭비지만 정확한 CBR.

근데 라이브에선 약간의 변동 허용이 일반적.

```bash
# 실전 라이브 (98% 가까운 CBR)
-b:v 6000k -maxrate 6000k -bufsize 12000k
```

`bufsize`는 보통 비트레이트의 1~2배. 작을수록 균일.

### 멀티 오디오 ABR

비디오 사다리 + 오디오 사다리 + 다국어 = 복잡한 매니페스트.

```bash
ffmpeg -i input.mkv \
  -filter_complex "[0:v]split=3[v1080][v720][v480]; \
                   [v720]scale=1280:720[v720s]; \
                   [v480]scale=854:480[v480s]" \
  \
  # 비디오 3단계
  -map "[v1080]" -c:v libx264 -b:v 6000k \
  -map "[v720s]" -c:v libx264 -b:v 3000k \
  -map "[v480s]" -c:v libx264 -b:v 1500k \
  \
  # 오디오 3개 (한/영/일)
  -map 0:a:0 -c:a aac -b:a 128k \
  -map 0:a:1 -c:a aac -b:a 128k \
  -map 0:a:2 -c:a aac -b:a 128k \
  \
  -f hls \
  -hls_time 6 \
  -hls_segment_type fmp4 \
  -master_pl_name master.m3u8 \
  -var_stream_map "\
    v:0,name:1080p \
    v:1,name:720p \
    v:2,name:480p \
    a:0,agroup:audio,language:ko,name:Korean,default:yes \
    a:1,agroup:audio,language:en,name:English \
    a:2,agroup:audio,language:ja,name:Japanese" \
  \
  'output/%v/playlist.m3u8'
```

`agroup`이 핵심. 같은 그룹 안의 오디오를 시청자가 선택.

마스터 결과:
```
#EXT-X-MEDIA:TYPE=AUDIO,GROUP-ID="audio",NAME="Korean",LANGUAGE="ko",DEFAULT=YES,URI="ko/playlist.m3u8"
#EXT-X-MEDIA:TYPE=AUDIO,GROUP-ID="audio",NAME="English",LANGUAGE="en",URI="en/playlist.m3u8"
#EXT-X-MEDIA:TYPE=AUDIO,GROUP-ID="audio",NAME="Japanese",LANGUAGE="ja",URI="ja/playlist.m3u8"

#EXT-X-STREAM-INF:BANDWIDTH=6128000,RESOLUTION=1920x1080,AUDIO="audio"
1080p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=3128000,RESOLUTION=1280x720,AUDIO="audio"
720p/playlist.m3u8

#EXT-X-STREAM-INF:BANDWIDTH=1628000,RESOLUTION=854x480,AUDIO="audio"
480p/playlist.m3u8
```

### Shaka Packager로 전환 — 분리 패턴

대규모는 인코딩과 패키징 분리. Shaka Packager가 표준.

**1단계: FFmpeg로 인코딩만**

```bash
# 1080p 인코딩
ffmpeg -i input.mp4 \
  -c:v libx264 -preset veryfast -tune zerolatency \
  -b:v 6000k -maxrate 6000k -bufsize 12000k \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 128k \
  -f mp4 1080p.mp4 &

# 720p 인코딩  
ffmpeg -i input.mp4 \
  -vf scale=1280:720 \
  -c:v libx264 -preset veryfast -tune zerolatency \
  -b:v 3000k -maxrate 3000k -bufsize 6000k \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 128k \
  -f mp4 720p.mp4 &

# 480p 인코딩
ffmpeg -i input.mp4 \
  -vf scale=854:480 \
  -c:v libx264 -preset veryfast -tune zerolatency \
  -b:v 1500k -maxrate 1500k -bufsize 3000k \
  -g 120 -keyint_min 120 -sc_threshold 0 \
  -c:a aac -b:a 96k \
  -f mp4 480p.mp4 &

wait
```

병렬 실행으로 시간 단축.

**2단계: Shaka로 패키징**

```bash
packager \
  'in=1080p.mp4,stream=video,init_segment=cmaf/1080p/init.mp4,segment_template=cmaf/1080p/seg_$Number$.m4s,playlist_name=1080p.m3u8,iframe_playlist_name=1080p-iframe.m3u8' \
  'in=1080p.mp4,stream=audio,init_segment=cmaf/audio/init.mp4,segment_template=cmaf/audio/seg_$Number$.m4s,playlist_name=audio.m3u8,hls_group_id=audio,hls_name=English' \
  'in=720p.mp4,stream=video,init_segment=cmaf/720p/init.mp4,segment_template=cmaf/720p/seg_$Number$.m4s,playlist_name=720p.m3u8' \
  'in=480p.mp4,stream=video,init_segment=cmaf/480p/init.mp4,segment_template=cmaf/480p/seg_$Number$.m4s,playlist_name=480p.m3u8' \
  \
  --hls_master_playlist_output master.m3u8 \
  --mpd_output manifest.mpd \
  --segment_duration 6 \
  --fragment_duration 6
```

한 번에 HLS + DASH 같이 생성. CMAF 세그먼트 공유.

### LL-HLS 패키징 — Shaka

LL-HLS는 FFmpeg로 거의 불가. Shaka 사용.

```bash
packager \
  'in=1080p.mp4,stream=video,init_segment=cmaf/1080p/init.mp4,segment_template=cmaf/1080p/seg_$Number$.m4s,playlist_name=1080p.m3u8' \
  \
  --hls_master_playlist_output master.m3u8 \
  --hls_playlist_type LIVE \
  --segment_duration 4 \
  --fragment_duration 0.2 \
  --low_latency_dash_mode \
  --utc_timings scheme_id_uri=urn:mpeg:dash:utc:http-iso:2014,value=https://time.akamai.com/?iso \
  \
  --preserved_segments_outside_live_window 5 \
  --time_shift_buffer_depth 30 \
  --minimum_update_period 2
```

핵심 옵션:
- `fragment_duration 0.2`: 200ms 부분 세그먼트 (Partial Segment)
- `low_latency_dash_mode`: chunked CMAF 활성화

이러면 LL-HLS 매니페스트에 `#EXT-X-PART` 태그 자동 생성.

### CMAF 통합 매니페스트

HLS와 DASH 동시 지원. 같은 .m4s 세그먼트 공유.

```bash
packager \
  'in=1080p.mp4,stream=video,init_segment=v_1080/init.mp4,segment_template=v_1080/seg_$Number$.m4s' \
  'in=720p.mp4,stream=video,init_segment=v_720/init.mp4,segment_template=v_720/seg_$Number$.m4s' \
  'in=480p.mp4,stream=video,init_segment=v_480/init.mp4,segment_template=v_480/seg_$Number$.m4s' \
  'in=1080p.mp4,stream=audio,init_segment=a/init.mp4,segment_template=a/seg_$Number$.m4s' \
  \
  --hls_master_playlist_output cmaf/master.m3u8 \
  --mpd_output cmaf/manifest.mpd \
  --segment_duration 6
```

결과 디렉토리:
```
cmaf/
├── master.m3u8        ← HLS 시청자가 받는 매니페스트
├── manifest.mpd       ← DASH 시청자가 받는 매니페스트
├── v_1080/
│   ├── init.mp4
│   └── seg_1.m4s, seg_2.m4s, ...   ← HLS와 DASH가 공유
├── v_720/
├── v_480/
└── a/
```

스토리지 절반. CDN 캐시 효율 극대화.

### Per-Stream HLS vs Single-Stream HLS

비디오와 오디오를 어떻게 분리할지.

**Per-Stream (분리)**:
```
v_1080/seg.m4s  (비디오만)
a/seg.m4s       (오디오만)
```

장점:
- 다국어 지원 (오디오 따로)
- ABR이 비디오만 전환
- CMAF의 기본

**Single-Stream (통합)**:
```
1080p/seg.ts  (비디오 + 오디오)
720p/seg.ts
```

장점:
- 단순
- 오래된 플레이어 호환성

치지직 같은 새 인프라는 Per-Stream. 모바일/구형 디바이스 케어 필요한 곳은 Single-Stream 유지.

### 3강 핵심 3줄

1. ABR Ladder는 모든 화질에 동일한 GOP (`-g -keyint_min -sc_threshold 0`)로 세그먼트 경계 정렬 필수
2. FFmpeg `-var_stream_map`으로 단일 명령 통합 vs Shaka Packager 분리 (대규모는 분리가 정석)
3. CMAF로 HLS+DASH 같은 .m4s 공유, LL-HLS는 Shaka의 `fragment_duration 0.2`로 Partial Segment 자동 생성
