## 레벨 3-3 / 2강: HDR과 색공간 — 4K HDR 콘텐츠 다루기

---

### HDR이 뭔지 실무 관점에서

기술 깊이 안 들어가고 결과만.

```
[SDR vs HDR 체감]
SDR: 일반 모니터, 밝기 최대 100 nits
HDR: 어두운 곳도 보이고 밝은 곳도 안 날아감, 1000+ nits

[시청자 경험 차이]
SDR: 해변 영상에서 하늘이 흰색으로 날아감
HDR: 하늘의 구름 디테일 보임, 모래의 그늘도 보임
```

영화관 같은 경험을 집에서. Netflix가 4K HDR로 차별화하는 이유.

---

### HDR 표준 4가지 — 어떤 거 다루나

```
[주요 HDR 표준]
HDR10: 가장 일반적, 무료
HDR10+: HDR10 확장, Samsung 주도
Dolby Vision: 최고급, 라이센스 유료
HLG: 방송용
```

실무에서:
- **HDR10**: 95% 경우. 기본 선택
- **HLG**: 라이브 방송 (BBC, NHK)
- **Dolby Vision**: 프리미엄 콘텐츠 (Netflix, Disney+)
- **HDR10+**: Samsung TV 위주

---

### HDR10 — 표준 HDR

가장 많이 쓰는 표준.

```
[HDR10 사양]
색공간: BT.2020
비트 깊이: 10비트
최대 밝기: 1000~4000 nits
메타데이터: 정적 (영상 전체에 한 번)
라이센스: 무료
```

YouTube, Netflix, Disney+, Apple TV+ 모두 지원.

가장 안전한 선택.

---

### HLG — 라이브 방송용

라이브에서 유일한 선택.

```
[HLG 특징]
방송 표준 (BBC + NHK 개발)
메타데이터 없음
SDR 자동 호환 (HDR TV는 HDR로, SDR TV는 SDR로)
실시간 처리 친화
```

```
[HDR10과 HLG 차이]
HDR10: 영상 전체 분석 후 메타데이터 생성 → VOD에 적합
HLG: 영상 자체에 HDR 정보 → 라이브에 적합
```

올림픽 같은 4K HDR 라이브 = HLG.

---

### Dolby Vision — 프리미엄

```
[Dolby Vision 특징]
10000 nits까지 (HDR10의 10배)
12비트 (HDR10의 4배 단계)
동적 메타데이터 (장면별)
라이센스 비용 큼
iPhone 카메라 녹화 지원
```

영화관 수준의 색 표현. Netflix 프리미엄 콘텐츠에서.

라이센스 비용 때문에 모든 콘텐츠에 안 씀.

---

### 10비트 색심도 — HDR의 필수 조건

```
[8비트 vs 10비트]
8비트: 256단계 (SDR 표준)
10비트: 1024단계 (HDR 필수)
```

왜 10비트가 필수냐?

HDR은 밝기 범위가 SDR의 10배. 8비트로 표현하면 단계가 굵어져서 그라데이션이 줄무늬처럼 보임 (Banding).

10비트는 4배 정밀해서 매끄러운 그라데이션.

---

### Color Space — 색공간

```
[색공간 비교]
sRGB / BT.709 (HD): 일반 모니터
DCI-P3: 영화관, 최신 폰
BT.2020 (4K HDR): HDR 전용
```

BT.2020이 더 짙은 빨강, 녹색 표현.

HDR 콘텐츠는 무조건 BT.2020.

---

### Chroma Subsampling — 4:2:0이 표준

```
[Chroma 표기]
4:4:4: Y, U, V 모두 같은 해상도 (영화 마스터)
4:2:2: U, V 가로 절반 (방송 장비)
4:2:0: U, V 가로/세로 절반 (스트리밍 표준)
```

스트리밍은 99% 4:2:0.

사람 눈이 색 변화에 둔감해서 절반 해상도로도 충분.

```bash
# Pixel format에서 표현
yuv420p: 8비트 4:2:0 (SDR)
yuv420p10le: 10비트 4:2:0 (HDR)
yuv422p: 8비트 4:2:2 (방송)
yuv444p: 8비트 4:4:4 (전문가)
```

### HDR10 인코딩 명령

VOD용 HDR10 인코딩.

```bash
ffmpeg -i input_master.mov \
  -c:v libx265 \
  -preset slow \
  -profile:v main10 \
  -pix_fmt yuv420p10le \
  -x265-params "colorprim=bt2020:transfer=smpte2084:colormatrix=bt2020nc:hdr-opt=1:repeat-headers=1:master-display='G(13250,34500)B(7500,3000)R(34000,16000)WP(15635,16450)L(10000000,1)':max-cll=1000,400" \
  -b:v 10000k \
  output_hdr10.mp4
```

복잡해 보이는 옵션 분해:
- `colorprim=bt2020`: 색공간 BT.2020
- `transfer=smpte2084`: PQ 곡선 (HDR10 표준)
- `colormatrix=bt2020nc`: 색 행렬
- `master-display`: 마스터링 모니터 정보
- `max-cll`: 최대 콘텐츠 밝기

---

### HLG 인코딩 명령 — 라이브용

라이브 HDR은 HLG.

```bash
ffmpeg -i input \
  -c:v hevc_nvenc \
  -preset p4 -tune ll \
  -profile:v main10 \
  -pix_fmt p010le \
  -x265-params "colorprim=bt2020:transfer=arib-std-b67:colormatrix=bt2020nc" \
  -b:v 8000k \
  -f flv rtmp://...
```

`arib-std-b67`가 HLG 표준.

`p010le`는 NVENC용 10비트 픽셀 포맷.

### 디바이스 호환성 확인 — 실무

시청자가 HDR 볼 수 있는지.

```
[HDR 시청 필수 조건]
1. HDR 콘텐츠
2. HDR 디스플레이 (TV, 모니터, 폰)
3. HDR 지원 플레이어
4. HDR 지원 케이블 (HDMI 2.0a 이상)
5. 운영체제 HDR 활성화
```

이 중 하나라도 빠지면 SDR 폴백.

실제 HDR 시청자 비율: < 10%.

---

### Tone Mapping — HDR을 SDR로

HDR 콘텐츠를 SDR 화면에서 어떻게 보여주나.

```
[Tone Mapping]
HDR 0~1000 nits → SDR 0~100 nits로 매핑

방법:
- Reinhard: 부드럽지만 콘트라스트 낮아짐
- Hable: 영화적 룩
- BT.2390: 표준 권장 (BBC 개발)
```

FFmpeg로 HDR → SDR:
```bash
ffmpeg -i hdr_input.mp4 \
  -vf "zscale=t=linear:npl=100,format=gbrpf32le,zscale=p=bt709,tonemap=tonemap=hable:desat=0,zscale=t=bt709:m=bt709:r=tv,format=yuv420p" \
  -c:v libx264 -b:v 6000k \
  sdr_output.mp4
```

복잡하지만 한 번 만들어두면 재사용.

### HDR 라이브 스트리밍의 현실

```
[HDR 라이브가 어려운 이유]
1. HDR 카메라 비쌈
2. HDR 인코딩 CPU/GPU 부담 ↑
3. 대역폭 30% 증가
4. ABR Ladder에 HDR + SDR 둘 다 만들어야
5. 시청자 디바이스 HDR 지원 적음
```

치지직 같은 일반 라이브엔 부담.

스포츠 중계, 콘서트 같은 프리미엄에서만.

---

### HDR ABR Ladder 예시

OTT가 운영하는 방식.

```
[Netflix 4K HDR 영화 - ABR]
HDR 트랙:
- 4K HDR10 12 Mbps
- 4K Dolby Vision 15 Mbps
- 1080p HDR10 6 Mbps

SDR 트랙 (폴백):
- 4K SDR 15 Mbps
- 1080p SDR 6 Mbps
- 720p SDR 3 Mbps
- 480p SDR 1.5 Mbps

오디오:
- Dolby Atmos
- 5.1 채널
- 스테레오
```

시청자 디바이스 + 가입 등급 + 네트워크 따라 자동 선택.

---

### 4K HDR 비트레이트 가이드

```
[해상도 vs HDR 비트레이트]
1080p SDR H.264: 6 Mbps
1080p HDR H.265: 4 Mbps
4K SDR H.265: 12 Mbps
4K HDR H.265: 15 Mbps (HDR이 30% 더)
4K HDR Dolby Vision: 25 Mbps
```

HDR이 같은 해상도에서 30% 더 비트레이트 필요.

10비트 처리 + 색공간 확장.

### 실무 결정 가이드

HDR 도입 여부.

```
[HDR 도입 OK]
✅ 프리미엄 OTT (Netflix, Disney+ 급)
✅ 영화/드라마 VOD
✅ 4K 콘텐츠 비중 큼
✅ 시청자 일부가 HDR TV 보유

[HDR 안 도입 OK]
❌ 게임 방송 라이브 (치지직)
❌ 광고 모델 무료 서비스
❌ 모바일 위주 (HDR 폰 적음)
❌ 1080p 위주 콘텐츠
```

치지직 = HDR 불필요. 게임이 4K HDR로 송출 안 되고 시청자도 모바일 위주.

---

### iPhone HDR 영상의 처리

iPhone Pro 모델이 Dolby Vision으로 녹화.

```
[iPhone HDR 영상 흐름]
iPhone 12 Pro+에서 4K HDR 녹화
→ Dolby Vision Profile 8.4 (Profile 5)
→ 파일 크기 큼

스트리밍에 올리려면:
1. Dolby Vision 그대로 (Netflix급만 가능)
2. HDR10으로 변환 (일반 OTT)
3. SDR로 다운컨버트 (호환성 우선)
```

대부분 SDR로 변환. iPhone HDR을 그대로 활용하는 플랫폼 적음.

```
"HDR은 콘텐츠 종류와 시청자 디바이스 분포로 결정합니다.

Netflix 같은 프리미엄 OTT는:
1. 영화/드라마가 4K HDR 마스터링됨

→ HDR10 + Dolby Vision 적극 활용

HDR 도입 시엔 HDR10이 기본, Dolby Vision은 라이센스 협상 가능한 곳만,
라이브는 HLG가 유일한 선택입니다."
```

### 2강 핵심 3줄

1. HDR10 (95% 경우), HLG (라이브), Dolby Vision (프리미엄) 세 표준 — 10비트 필수 (Banding 방지), BT.2020 색공간, yuv420p10le 픽셀 포맷
2. HDR이 같은 해상도에 30% 더 비트레이트, 시청 조건 5가지 (콘텐츠/디스플레이/플레이어/케이블/OS) 모두 만족해야 — 실제 HDR 시청자 < 10%
3. 일반 라이브 (치지직)는 HDR 불필요, 프리미엄 OTT (Netflix급)는 HDR + Dolby Vision으로 차별화 — 라이브 HDR은 HLG 외 선택지 없음
