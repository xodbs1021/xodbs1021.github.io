## 레벨 3-6 / 2강: 코덱 트랜스코딩 비용 — 실무 계산

---

### 트랜스코딩 비용을 왜 알아야 하나

라이브 플랫폼 운영의 가장 큰 비용 항목 중 하나.

```
[라이브 플랫폼 비용 분포]
CDN: 60~70%
트랜스코딩 (서버): 15~20%
스토리지: 5~10%
시그널링/기타: 5%
```

CDN 다음으로 트랜스코딩이 큼.

코덱 선택이 직접 영향. 면접에서 비용 답변 필수.

---

### CPU vs GPU 트랜스코딩 비용

기본 비교.

```
[1080p60 H.264 트랜스코딩 - 한 채널]

x264 medium (CPU):
- CPU 부하: 한 코어 100%
- 서버당 채널: 4~6개
- 비용: 시간당 약 $0.30

x264 veryfast (CPU - 라이브):
- 한 코어 약 60%
- 서버당 채널: 10~15개
- 비용: 시간당 약 $0.10

NVENC (GPU):
- GPU 부하: 5%
- 서버당 채널: 50~100개
- 비용: 시간당 약 $0.02
```

GPU가 15배 효율적. 대규모 라이브 = GPU 필수.

---

### 코덱별 GPU 부하

같은 NVENC라도 코덱 따라 다름.

```
[NVIDIA L40 한 대 - 1080p60 인코딩]
H.264 NVENC: 100채널 동시
H.265 NVENC: 80채널 (Pascal+)
AV1 NVENC: 50채널 (Ada Lovelace만)

VP9 NVENC: 없음 (CPU 만)
```

H.264가 가장 효율.

H.265는 약간 부담.

AV1은 가장 부담 + 최신 GPU 필요.

### 자체 인프라 비용 비교

NVIDIA L40 서버 직접 운영.

```
[L40 서버 운영 비용]
서버 구매: $40,000 (5년 상각)
연 운영비: $5,000 (전기, 네트워크)
연 총: $13,000

처리 능력:
- 1080p H.264 100채널 동시
- 24시간 운영

1000채널 24/7 운영:
- 서버 10대 필요
- 연 $130,000

vs AWS MediaConvert:
- 연 $20,000,000+ (1.5만배 차이!)
```

자체 인프라가 압도적으로 싸다.

이래서 치지직, 트위치, YouTube 같은 곳은 자체 GPU.

### 코덱별 인코딩 비용 - 종합

```
[1000 채널 1년 트랜스코딩 비용]

H.264 NVENC 자체 인프라:
- 서버 10대 × $13K = $130K
- 매우 저렴

H.265 NVENC 자체 인프라:
- 서버 13대 × $13K = $170K
- 중간

AV1 NVENC 자체 인프라:
- 서버 20대 × $13K = $260K (RTX 4000+ 비쌈)
- 약간 비쌈

H.265 CPU (x265):
- 서버 200대 × $13K = $2.6M
- 매우 비쌈

CPU만으로 4K AV1:
- 거의 불가능 (서버 수천 대)
```

GPU는 필수. CPU만으론 대규모 운영 불가능.

### ABR Ladder 비용 - 같은 채널에 여러 화질

ABR 운영 시 한 채널이 GPU에 미치는 부담.

```
[1채널 ABR Ladder]
입력: 1080p 6 Mbps
출력:
- 1080p H.264 6 Mbps
- 720p H.264 3 Mbps
- 480p H.264 1.5 Mbps
- 360p H.264 600 Kbps

GPU 부하 (NVENC):
- 1080p 인코딩: 5%
- 720p: 3%
- 480p: 2%
- 360p: 1%
- 합계: 11%

한 채널이 GPU 11% 사용
1 GPU 9채널 동시 가능
```

---

### CDN 비용과 코덱의 관계

코덱이 CDN 비용에 미치는 영향.

```
[1000 채널 100만 시청자 일평균 4시간]

H.264 1080p 6 Mbps:
- 트래픽: 1000 × 100만 × 6Mbps × 4시간 = 약 11 PB/일
- CDN 비용: $0.05/GB × 11M GB = $550K/일

H.265 1080p 3 Mbps (50% 절감):
- 트래픽: 5.5 PB/일
- CDN 비용: $275K/일
- 일 $275K 절감

연 $100M 절감 가능!
```

이게 Netflix가 H.265 적극 채택하는 이유.

근데 라이센스 비용 + 트랜스코딩 비용 + 호환성 이슈 고려해야.

---

### 코덱 효율 vs 라이센스 비용 계산

```
[H.264 → H.265 전환 검토 - 100만 디바이스]

CDN 절감: 연 $100M
H.265 라이센스: 연 $30M
H.265 인코딩 추가 GPU: 연 $5M
호환성 폴백 (H.264 동시 운영): 연 $10M

순 절감: 100 - 30 - 5 - 10 = $55M
→ 전환 가치 있음

[같은 계산 - AV1]
CDN 절감: 연 $130M (H.265보다 큼)
AV1 라이센스: $0
AV1 인코딩 추가 GPU: 연 $10M (더 비싼 GPU)

순 절감: 130 - 0 - 10 = $120M
→ AV1이 더 유리
```

AV1이 결국 답이 되는 이유.

### 한국 라이브 플랫폼의 비용 현실

치지직급 가상 분석.

```
[가상 시나리오 - 한국 라이브]
동시 라이브 채널: 5000개
평균 시청자/채널: 100명
피크 시청자: 50만

[비용]
CDN: 연 200억
트랜스코딩: 연 30억
스토리지: 연 10억
인프라/운영: 연 50억

[H.265 전환 검토]
CDN 절감: 50% × 200억 = 100억
H.265 라이센스: 협상가 10~30억
인코딩 GPU 추가: 5억

순 절감: 65~85억

근데:
- 호환성 문제로 H.264 폴백 운영
- 운영 복잡도 증가
- 실제 절감 약 30~50억

[결론]
H.264 유지가 안정적
H.265 도입 시 ROI 확실하지 않음
```

이게 한국 플랫폼이 H.265 도입 천천히 하는 이유.

### 인코더 선택의 실무 비용 영향

같은 H.264여도 인코더 선택.

```
[같은 1080p60 H.264 출력]

x264 ultrafast: CPU 30%, 화질 낮음
x264 veryfast: CPU 60%, 화질 보통 (라이브 표준)
x264 medium: CPU 100%, 화질 좋음 (VOD)
x264 slow: 매우 느림

NVENC p1: GPU 2%, 화질 낮음
NVENC p4: GPU 5%, 화질 좋음 (라이브 표준)
NVENC p7: GPU 20%, 화질 매우 좋음
```

라이브는 veryfast 또는 NVENC p4.
VOD는 medium 또는 NVENC p7.

### Per-Title Encoding의 비용 효과

Netflix 같은 곳의 절감 방식.

```
[고정 ABR Ladder]
모든 영화 4K 12 Mbps
저장: 영화당 약 7 GB

[Per-Title]
정적 영화: 4K 6 Mbps
액션 영화: 4K 18 Mbps
평균: 4K 8 Mbps
저장: 영화당 평균 4.7 GB (33% 절감)

[100만 영화 × 5년 보관]
고정: 7 PB
Per-Title: 4.7 PB
S3 비용 차이: 연 약 $80K
```

저장 비용도 코덱 결정에 영향.

### 면접 답변 - 비용 분석

"코덱 비용 어떻게 계산하세요?" 질문.

```
"세 가지 비용을 고려합니다:

1. 트랜스코딩 비용
   - 자체 GPU 인프라: 연 십수억
   - 클라우드 (AWS): 연 수십~수백억
   - 자체 GPU가 압도적 절감

2. CDN 비용
   - 코덱 효율로 결정
   - H.265는 H.264 대비 50% 절감
   - AV1은 H.264 대비 60% 절감

3. 라이센스 비용
   - H.264: 협상 가능, 소규모 면제
   - H.265: 여러 풀, 비싸고 복잡
   - AV1: 무료

실무 분석:
- 한국 라이브 1000 채널 운영
- H.264 유지: 연 약 250억
- H.265 전환: 연 약 200억 (CDN 절감 - 라이센스 - 추가 인프라)
- 실제 절감: 50억
- 위험: 호환성 이슈, 운영 복잡도

ROI가 확실하지 않아 H.264 유지가 안정적입니다.
다음 코덱은 AV1을 검토할 예정입니다."
```

### 2강 핵심 3줄

1. 트랜스코딩 비용은 CPU 대비 GPU가 15배 효율, 자체 인프라는 클라우드 대비 1.5만배 절감 — NVENC가 라이브 트랜스코딩 표준
2. CDN 절감 (H.265 50%, AV1 60%) vs 라이센스 + 인코딩 추가 비용 - 호환성 폴백 = 순 ROI 계산 필수
3. Per-Title Encoding (Netflix)로 영화별 ABR 최적화 33% 절감, Twitch는 작은 스트리머 ABR 제외로 비용 절감 — H.264 유지가 안정적인 경우 많음
