### WebRTC 앱에서 Opus 협상

브라우저 코드.

```javascript
const pc = new RTCPeerConnection({
  iceServers: [{ urls: 'stun:stun.l.google.com:19302' }]
});

// 오디오 트랙 추가
const stream = await navigator.mediaDevices.getUserMedia({
  audio: {
    echoCancellation: true,
    noiseSuppression: true,
    autoGainControl: true,
    sampleRate: 48000,
  },
  video: false,
});

stream.getTracks().forEach(track => pc.addTrack(track, stream));

// SDP에서 Opus 우선 순위 설정 (선택)
const offer = await pc.createOffer();
const munged = preferOpus(offer.sdp);
await pc.setLocalDescription({ type: 'offer', sdp: munged });
```

`getUserMedia`의 `echoCancellation` 등은 WebRTC 내장 기능 활성화. 통화 품질의 핵심.

### SDP에서 Opus 옵션 조작

```javascript
function tweakOpus(sdp) {
  // Opus 페이로드 타입 찾기 (보통 111)
  const opusPt = sdp.match(/a=rtpmap:(\d+) opus\/48000/)[1];
  
  // fmtp 라인 추가/수정
  const fmtpLine = `a=fmtp:${opusPt} minptime=10;useinbandfec=1;usedtx=1;stereo=1;maxaveragebitrate=128000`;
  
  // 기존 fmtp 라인 교체 또는 추가
  if (sdp.includes(`a=fmtp:${opusPt}`)) {
    sdp = sdp.replace(/a=fmtp:111[^\r\n]+/, fmtpLine);
  } else {
    sdp = sdp.replace(
      `a=rtpmap:${opusPt} opus/48000/2`,
      `a=rtpmap:${opusPt} opus/48000/2\r\n${fmtpLine}`
    );
  }
  
  return sdp;
}
```

`fmtp` 옵션:
- `minptime=10`: 최소 10ms 프레임
- `useinbandfec=1`: FEC 활성화
- `usedtx=1`: 침묵 시 전송 중단
- `stereo=1`: 스테레오
- `maxaveragebitrate=128000`: 최대 128 Kbps

### Discord 봇 — Opus 직접 다루기

Discord.js에서 음성 봇.

```javascript
const { Client, GatewayIntentBits } = require('discord.js');
const { joinVoiceChannel, createAudioPlayer, createAudioResource } = require('@discordjs/voice');

const client = new Client({
  intents: [
    GatewayIntentBits.Guilds,
    GatewayIntentBits.GuildVoiceStates,
  ],
});

client.on('messageCreate', async (message) => {
  if (message.content === '!play') {
    const channel = message.member.voice.channel;
    
    const connection = joinVoiceChannel({
      channelId: channel.id,
      guildId: channel.guild.id,
      adapterCreator: channel.guild.voiceAdapterCreator,
    });
    
    const player = createAudioPlayer();
    
    // Opus 오디오 파일 재생
    const resource = createAudioResource('./music.opus');
    player.play(resource);
    
    connection.subscribe(player);
  }
});
```

Discord는 내부적으로 모두 Opus. MP3, WAV 입력해도 Opus로 변환 후 전송.

### 음악 봇 — MP3 → Opus 스트리밍

YouTube 음악 봇 같은 경우.

```javascript
const { exec } = require('child_process');
const { createAudioResource, StreamType } = require('@discordjs/voice');

function playFromYoutube(url, connection) {
  // youtube-dl로 받고 FFmpeg로 Opus 변환
  const ffmpeg = exec(`youtube-dl -o - "${url}" | ffmpeg -i pipe:0 -c:a libopus -b:a 96k -f opus pipe:1`);
  
  const resource = createAudioResource(ffmpeg.stdout, {
    inputType: StreamType.OggOpus,
  });
  
  const player = createAudioPlayer();
  player.play(resource);
  connection.subscribe(player);
}
```

Opus가 Discord 표준이라 변환 효율적.

### 모바일 SDK - iOS Opus

iOS WebRTC SDK에서.

```swift
import WebRTC

let factory = RTCPeerConnectionFactory()

// 오디오 설정
let audioConstraints = RTCMediaConstraints(
  mandatoryConstraints: [
    "echoCancellation": "true",
    "googNoiseSuppression": "true",
    "googAutoGainControl": "true",
  ],
  optionalConstraints: nil
)

let audioSource = factory.audioSource(with: audioConstraints)
let audioTrack = factory.audioTrack(with: audioSource, trackId: "audio0")

let pc = factory.peerConnection(
  with: RTCConfiguration(),
  constraints: audioConstraints,
  delegate: self
)

pc.add(audioTrack, streamIds: ["stream0"])
```

iOS WebRTC는 내부 Opus 자동 사용. 개발자가 코덱 명시 안 해도 됨.

### Android SDK - Opus

Android에서 WebRTC.

```kotlin
import org.webrtc.*

val factory = PeerConnectionFactory.builder()
  .createPeerConnectionFactory()

// 오디오 트랙 생성
val audioConstraints = MediaConstraints()
audioConstraints.mandatory.add(
  MediaConstraints.KeyValuePair("echoCancellation", "true")
)

val audioSource = factory.createAudioSource(audioConstraints)
val audioTrack = factory.createAudioTrack("audio0", audioSource)

val pcConfig = PeerConnection.RTCConfiguration(emptyList())
val pc = factory.createPeerConnection(pcConfig, pcObserver)

pc?.addTrack(audioTrack, listOf("stream0"))
```

Java/Kotlin도 비슷. Opus가 표준이라 코덱 명시 불필요.

### Opus → AAC 변환 - 실무

WebRTC에서 받은 Opus를 HLS로 배포할 때.

```bash
# 단순 변환
ffmpeg -i input.opus -c:a aac -b:a 128k output.m4a

# 영상 + Opus 오디오 → HLS (AAC)
ffmpeg -i webrtc_recording.webm \
  -c:v copy \
  -c:a aac -b:a 128k -ar 48000 \
  -hls_time 6 -hls_list_size 5 \
  -hls_flags delete_segments \
  output.m3u8
```

영상은 copy로 빠르게, 오디오만 트랜스코딩.

### 음성 인식 — Whisper에 Opus 입력

음성 인식 모델에 Opus 직접 입력.

```python
import whisper

# Opus 파일도 자동 디코딩
model = whisper.load_model("base")
result = model.transcribe("audio.opus")
print(result["text"])
```

Whisper는 FFmpeg 사용. Opus, MP3, WAV 모두 자동 처리.

성능 차이:
- WAV 원본: 가장 정확
- Opus 128 Kbps: 거의 차이 없음
- Opus 32 Kbps: 약간 정확도 떨어짐
- Opus 8 Kbps: 명확한 정확도 감소

### PokeClip의 음성 분석에 Opus?

가상 시나리오.

```
[현재 PokeClip - AAC]
스트리머 → MPEG-TS (SRT) → AAC 트랙 4개
       → 분석 서버 → Whisper 입력

[대안 - Opus 사용 가능?]
스트리머 → WebRTC (WHIP) → Opus 4개
        → 분석 서버 → 디코딩 → Whisper

이론적으론 OK
실제로는:
- MPEG-TS의 멀티 트랙 PID가 더 안정적
- SRT가 신뢰성 우수
- AAC도 음성 인식 충분
→ AAC + MPEG-TS 유지가 맞음
```

Opus 장점이 결정적이지 않으면 변경 안 함.

  48 Kbps + 5ms 프레임

### Opus 모니터링 - WebRTC Stats API

브라우저에서 Opus 통계.

```javascript
async function getOpusStats(pc) {
  const stats = await pc.getStats();
  
  stats.forEach(stat => {
    if (stat.type === 'outbound-rtp' && stat.kind === 'audio') {
      console.log('Outbound Audio:');
      console.log('  Codec:', stat.codecId);
      console.log('  Packets sent:', stat.packetsSent);
      console.log('  Bytes sent:', stat.bytesSent);
      console.log('  Bitrate:', stat.targetBitrate);
    }
    
    if (stat.type === 'inbound-rtp' && stat.kind === 'audio') {
      console.log('Inbound Audio:');
      console.log('  Packets received:', stat.packetsReceived);
      console.log('  Packets lost:', stat.packetsLost);
      console.log('  Jitter:', stat.jitter);
    }
  });
}

// 주기적 측정
setInterval(() => getOpusStats(pc), 1000);
```

### 흔한 실전 문제

WebRTC + Opus 환경.

```
[문제 1: 에코]
원인: echoCancellation 비활성화
해결: getUserMedia 설정에서 echoCancellation: true

[문제 2: 음질 떨어짐]
원인: 비트레이트 부족
해결: SDP fmtp에 maxaveragebitrate 명시

[문제 3: 모노만 들림]
원인: stereo 옵션 미설정
해결: SDP에 stereo=1, sprop-stereo=1 추가

[문제 4: iOS Safari에서 안 됨]
원인: Opus 옵션 호환성
해결: iOS는 기본 설정 사용, 옵션 안 건드림

[문제 5: 손실 환경에서 끊김]
원인: FEC 비활성화
해결: useinbandfec=1 명시
```

### 2강 핵심 3줄

1. WebRTC 앱은 `echoCancellation/noiseSuppression` 활성화 + Opus SDP fmtp 옵션 (`useinbandfec`, `usedtx`, `stereo`, `maxaveragebitrate`) — 모바일 SDK는 자동 처리
2. Discord/Zoom/Whisper 같은 다양한 환경에서 활용, Opus → AAC 변환은 FFmpeg로 영상은 copy + 오디오만 트랜스코딩으로 빠르게
3. 실전 문제는 에코, 비트레이트 부족, 모노 강제, FEC 비활성화 — `chrome://webrtc-internals`로 실시간 통계 모니터링
