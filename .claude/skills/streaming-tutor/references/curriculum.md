## 레벨 1: 스트리밍 기초 개념 ⭐ (선행 필수, 추가)

아무것도 모를 때 가장 먼저 배워야 할 토대. 이걸 알아야 이후 프로토콜/코덱이 왜 존재하는지 이해됨.

| ID | 주제 | 핵심 질문 |
|----|------|---------|
| 1-1 | 인터넷 동영상 전송 원리 | 유튜브 영상이 어떻게 내 화면에 오는가? |
| 1-2 | 비트레이트 · 해상도 · FPS | 화질이 좋다는 게 정확히 무슨 뜻인가? |

## 레벨 2: 스트리밍 프로토콜 (멘토 목록)

어떤 방식으로 영상을 전송하는가. 각 프로토콜의 특징과 용도를 이해.

| ID | 주제 | 특징 | HiPick 연관 |
|----|------|------|------------|
| 2-1 | RTMP | 가장 오래된 방식, OBS로 송출할 때 씀 | 스트리머가 방송할 때 사용 |
| 2-2 | HLS | 가장 많이 쓰임, 시청자가 볼 때 | 저장된 클립 배포에 사용 |
| 2-3 | DASH | HLS 경쟁자, 유튜브/넷플릭스 | HLS 이해 후 비교 |
| 2-4 | LL-HLS | 저지연 HLS (2-5초) | 라이브 스트림 저지연 배포 |
| 2-5 | SRT | 불안정한 네트워크에 강함 | 현장 방송 중계 |
| 2-6 | WebRTC | 초저지연 (0.5초 미만) | 실시간 상호작용 |
| 2-7 | WHIP/WHEP | WebRTC 표준화된 최신 방식 | 차세대 라이브 방송 |

## 레벨 3: 비디오/오디오 코덱 (멘토 목록)

영상을 어떻게 압축하는가. 코덱 선택이 화질과 파일 크기를 결정.

| ID | 주제 | 특징 |
|----|------|------|
| 3-1 | H.264 / AVC | 가장 범용, 모든 기기에서 재생됨 |
| 3-2 | AAC | 오디오 표준, MP3보다 좋음 |
| 3-3 | H.265 / HEVC | H.264 대비 절반 크기, 더 선명 |
| 3-4 | Opus | WebRTC 기본 오디오 코덱 |
| 3-5 | AV1 | 오픈소스, 유튜브 기본값 |
| 3-6 | VP9 | 구글이 만든 코덱 |

| ID | 주제 | 핵심 내용 |
|----|------|---------|
| 4-1 | FFmpeg 기초 | 설치 → 기본 변환 명령어 |
| 4-2 | ABR ladder 설계 | 여러 화질 버전을 동시에 만드는 법 |
